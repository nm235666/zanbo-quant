# Frontend Change Checklist

适用于页面、组件、图表、路由、权限、交互、前端数据接入类改动。

## 变更前

- 这个任务是否直接服务前端展示业务？
- 是否已有现成的 page / store / API client / shared 组件可复用？
- 是否会影响路由、权限、导航入口？
- 是否会影响现有 API 字段兼容性？
- 是否真的需要改后端，还是前端即可完成？

## 变更中

- 优先做最小可见改动
- 保持 loading / empty / error 状态可用
- 保持现有样式和交互体系一致
- 通过 stores 管理状态，不要绕过
- 不顺手做无关重构

## 变更后

- 页面能否正常展示？
- 数据能否正常加载？
- 空态和报错态是否合理？
- 路由和权限是否仍正确？
- 是否需要同步更新文档？
