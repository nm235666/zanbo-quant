# Scheduler Parity Checklist

用于核对 cron 与 job_orchestrator 的一致性，避免“定义了但没跑”或“跑了旧命令”。

## 定义一致性

- `job_registry.py` 中关键任务是否存在？
- 定时表达式是否与预期时区一致（UTC/中国时间）？
- run script 是否仍指向最新命令参数？

## 安装一致性

- `install_all_crons.sh` 是否包含对应任务？
- `crontab -l` 是否包含并生效？
- 是否有重复安装导致双跑？

## 运行一致性

- 最近一次 run_id 状态是否成功？
- 是否有常驻失败任务未告警？
- 是否存在锁文件僵死导致任务不跑？

## 故障处理

- 失败是否有明确分类（配置/源站/代码）？
- 是否有回填计划和补跑命令？
- 是否需要临时降频或限流？
