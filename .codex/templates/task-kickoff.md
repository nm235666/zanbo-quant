# Task Kickoff Template

用于明确单个任务，避免范围漂移。

```text
任务目标：
[只写一个明确目标]

上下文范围：
只关注这些目录/模块：
- ...
- ...

硬约束：
- 不改无关文件
- 不引入新依赖
- 不改公开接口
- 改动最小且可交付（每轮 2-4 个强相关改动）
- 优先复用现有实现

请先不要直接写代码，先输出：
1. 你理解的任务
2. 涉及的文件和模块
3. 风险点
4. 最小实施计划

如果这次任务只分析不写代码，保留以上结构继续分析，不要进入实施。

执行规则：
- 未经确认，不要跨模块重构
- 一次执行一个最小可交付批次（2-4 个强相关改动）
- 每一步结束后汇报改动摘要、验证结果、剩余风险
- 如果发现范围扩大，立即停止并说明原因

额外要求：
- 优先服务前端展示业务
- 兼顾测试可用性
- 如改动影响文档，需同步更新
```
