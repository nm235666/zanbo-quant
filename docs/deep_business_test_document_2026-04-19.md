# 《深度业务测试文档》

## 1. 测试概述
- 测试目标：以 `docs/project_final_target_shape_dual_horizon_2026-04-19.md` 为唯一基准，审计系统是否已达到“日度可执行的双周期投资操作台”完成态。
- 测试对象：`http://192.168.5.52:8077`（admin/pro/limited 三类账号）。
- 测试方式：
  - 浏览器真实访问与真实操作（登录、跳转、筛选、重复提交、受限访问、跨页回查）。
  - Playwright 自动化回归（`mainline + stocks + signals + research + navigation + auth + system + smoke`）。
  - 浏览器会话下同源 API 探测（携带真实登录 token，验证执行承接、幂等、追溯链路）。
- 测试范围：前端业务链路与权限行为、状态流转、决策闭环、长线宏观链路、指标门禁现状。
- 参考基准：`project_final_target_shape_dual_horizon_2026-04-19.md`。
- 说明：本次关注“完成态差距审计”，不是普通可用性走查。

## 2. 完成态基准拆解
本次采用以下验收框架：
1. 双周期投资操作台目标（短线可执行 + 长线可应对）。
2. `3 分钟判断 SLA`（是否快速回答6问）。
3. `5 分钟动作 SLA`（是否形成首个可执行动作）。
4. 三层股池模型（系统机会池/用户关注池/用户持仓池）及流转真实性。
5. 仓位口径统一（账户整体仓位占比，而非孤立数量）。
6. 短长线冲突裁决（系统必须给结论）。
7. 长线实体页面能力（`/macro/regime`、`/portfolio/allocation`）
8. 每日工作流 7 步是否成立。
9. 核心信息架构 9 页职责是否稳定承载。
10. 短线能力完整性（主题TopN、动作卡、承接、追踪、回查）。
11. 长线能力完整性（三周期状态机、组合动作映射、冲突裁决、长线复盘）。
12. 决策闭环完整性（输入→分析→判断→动作→执行→跟踪→反馈→复盘→修正）。
13. 数据治理与权限治理（规则可见、状态一致、幂等、受限解释）。
14. 验收指标相关能力（SLA、主入口覆盖、追溯率、复盘率、门禁阻断）。

## 3. 核心业务链路识别
1. 链路：研究工作台主入口链
- 完成态职责：唯一主入口，承载“今日6问”和主线分发。
- 对应页面：`/research/workbench`。
- 验证方式：登录落点验证 + mainline 回归 + 页面元素核验。
- 结论：**部分成立**（入口已收敛，但后续执行闭环未完全打通）。

2. 链路：短线主题 -> 标的决策 -> 仓位建议
- 完成态职责：形成“买/卖/仓位/失效/优先级”动作卡。
- 对应页面：`/research/workbench`、`/research/decision`、`/signals/themes`。
- 验证方式：页面控件与字段验证 + 决策动作提交探测。
- 结论：**部分成立**（字段存在，但动作到执行承接与回查不稳定）。

3. 链路：决策动作 -> 执行队列 -> 执行看板
- 完成态职责：动作自动承接执行任务，状态全链可追踪。
- 对应页面：`/research/decision`、`/portfolio/orders`。
- 验证方式：提交动作后查询 `decision/actions` 与 `portfolio/orders` 关联。
- 结论：**不成立**（实际观测到 orders 为 0，且 pro API 权限拒绝）。

4. 链路：执行结果 -> 复盘 -> 规则修正
- 完成态职责：可回填、可追溯、可回放。
- 对应页面：`/portfolio/review`、`/macro/regime`。
- 验证方式：复盘页展示与宏观 outcome 录入能力核验。
- 结论：**部分成立**（有录入能力，但与执行链路联动证据不足）。

5. 链路：宏观三周期 -> 组合配置 -> 冲突裁决
- 完成态职责：长线状态变化自动映射组合动作，并给短长线冲突裁决。
- 对应页面：`/macro/regime`、`/portfolio/allocation`。
- 验证方式：角色访问测试 + 页面内容与裁决展示验证。
- 结论：**部分成立**（admin 可访问；pro 被拦截导致研究主角色不可用）。

6. 链路：受限访问 -> 升级解释 -> 替代路径
- 完成态职责：受限场景 100% 给出受限任务、缺失权限、替代入口。
- 对应页面：`/upgrade?from=...`。
- 验证方式：limited 多入口受限访问测试。
- 结论：**部分成立**（大多入口符合，但存在入口行为不一致）。

## 7. 隐藏规则与系统机制判断

5. 决策动作可写入，但未必形成执行订单。

2. 权限矩阵可能存在路由白名单漂移，导致 limited 对部分新页放行。
3. 执行订单链路可能受后端运行版本、表状态或权限策略共同影响。

## 9. 优先级排序与修复建议
### Top 10 问题

- 影响主链：决策->执行->跟踪。

2. action->order 全链不可回查（致命）
- 影响主链：执行追踪与复盘。
- 类型：结构性问题。

4. 幂等键失效导致重复动作（高）
- 影响主链：动作质量与数据可信度。
- 类型：结构性问题。

5. 页面可访问但 API 权限拒绝（高）
- 影响主链：执行看板可用性。
- 类型：结构性问题。

6. 受限入口覆盖不一致（高）
- 影响主链：权限透明度与用户预期。
- 类型：短期可修。

7. 完成态 SLA 门禁缺失（高）
- 影响主链：验收与发布治理。
- 类型：结构性问题。

8. 复盘链路与执行结果关联弱（中）
- 影响主链：闭环学习能力。
- 类型：结构性问题。

9. 冲突裁决展示弱（中）
- 影响主链：短长线协同决策。

10. TopN 个性化仅本地持久化（中）
- 影响主链：跨设备一致性。
- 类型：短期可修。

### 若只能先修 3 个，建议顺序
1. **动作自动承接执行 + 全链回查（1+2合并）**：不修则完成态闭环无法成立。
2. **Pro 长线核心页权限修复（3）**：不修则双周期目标失真。
3. **幂等与一致性修复（4+5）**：不修则数据可信度与执行稳定性持续恶化。

## 10. 总体结论
1. 当前系统是否已接近“日度可执行的双周期投资操作台”：**部分接近，但尚未达标**。
2. 当前最大问题：**不是页面缺失，而是执行承接与闭环可靠性不足 + 权限与规则一致性不足**。
3. 系统形态判断：**更像“能力已展示、闭环待固化”的业务骨架，而非可长期稳定运行的完成态产品**。
4. 是否建议进入下一阶段：**有条件进入**，前提是先补齐 P0 闭环（自动承接、回查、关键权限一致性、幂等）。
5. 总体判断：**有条件通过**。
- 理由：核心页面与主入口体系已具备，但完成态关键验收项（执行承接、全链追溯、SLA门禁）未达要求，暂不满足“完全通过”。
