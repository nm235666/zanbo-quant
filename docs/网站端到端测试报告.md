## 三、功能异常清单（不能使用/报错/无效）

### 3.1 数据源监控 - 数据全零（严重）

| 项目 | 实际值 | 预期值 |
|------|--------|--------|
| 数据源正常 | 0 | 应有实际数据源数量 |
| 数据源延迟 | 0 | 应有延迟监控 |
| 数据源异常 | 0 | 应有异常监控 |
| 进程正常 | 0 | 应有进程监控 |
| 进程告警 | 0 | 应有告警监控 |
| 进程异常 | 0 | 应有异常监控 |
| 任务定义 | 0 | 应与任务调度中心一致（48个） |
| 最近运行 | 0 | 应有运行记录 |
| Worker状态 | 离线 | 应在线 |

**问题分析**：数据源监控页面所有指标为0，Worker显示离线，说明监控采集链路未正常运行。虽然任务调度中心显示成功率97.5%，但数据源监控与任务调度中心的数据未打通。

**影响**：运维人员无法通过监控页面判断系统真实健康状态，需手动查看日志。

### 3.2 实时连接 - 频繁重连（中等）

**现象**：多个页面顶部状态栏显示"实时连接：重连中"或"在线"，状态不稳定。

**影响**：WebSocket实时推送功能不稳定，可能导致数据更新延迟。

## 四、体验问题清单（能点但体验差/路径绕/反馈不清晰）

### 4.1 决策看板 - 裁决精准度为0%（严重）

**现象**：决策看板显示"看多命中率 0%"、"看空命中率 0%"、"整体准确率 0%"，但近期首席裁决显示多条记录。

**问题分析**：
- 近期首席裁决有5条记录（均为000001.SZ），但精准度统计为0%
- 可能原因：1) 统计逻辑有bug；2) 需要时间窗口计算（当前未到评估时点）；3) 裁决结果未与后续股价走势关联

**影响**：用户无法评估决策质量，"裁决精准度"模块失去参考价值。

### 4.2 市场结论 - 可信度0%（中等）

**现象**：市场结论显示"结论可信度 0%"，提示"当前可信度较低，建议人工复核"。

**问题分析**：
- 市场结论的数据更新时间为2026/4/21 21:55:18，数据是新鲜的
- 但可信度为0%，说明结论生成逻辑可能未完成或数据不完整
- 主要风险显示"当前缺少显式风险情景或风险状态数据"

**影响**：用户无法信任系统自动生成的市场结论，需要完全依赖人工判断。

### 4.3 候选漏斗 - 全流程为空（中等）

**现象**：候选漏斗所有阶段（已进入、已增强、AI初筛、短名单、待决策、已确认、已淘汰、已暂缓、已执行、已复盘）均为0。

**问题分析**：
- 评分总览显示有8只自动短名单股票（如旺能环境、申能股份等）
- 信号总览显示203条信号，其中23条活跃
- 但候选漏斗中没有任何候选标的

**影响**：候选漏斗作为"候选生命周期状态机"，无法与实际信号/评分数据打通，形成数据孤岛。

### 4.4 执行链路 - 全为空（中等）

**现象**：
- 计划单：0条订单
- 持仓看板：0持仓
- 执行复盘：0条复盘记录

**问题分析**：
- 虽然系统有评分、信号、候选方向，但没有形成从"候选"到"下单"到"持仓"到"复盘"的完整闭环
- 用户可能需要在系统外完成实际交易操作

**影响**：执行链路断裂，系统定位为"研究工具"而非"交易管理工具"。

### 4.5 多角色研究 - 空状态提示不足（轻微）

**现象**：多角色研究页面所有区域显示"尚未启动"、"暂无数据"、"等待任务返回"。

**问题分析**：
- 页面缺少引导用户发起首次分析的提示
- 各区域状态提示过于技术化（如"idle"、"等待解析"）

**建议**：增加新手引导，提示用户输入股票代码发起分析。

## 五、数据展示问题清单

### 5.1 重复裁决记录

**现象**：近期首席裁决显示5条记录，其中4条是同一只股票（000001.SZ）在相近时间（04-19 06:37-06:39）的重复提交，标记为"深测-重复提交"。

**问题分析**：
- 系统未对重复提交进行有效去重
- 测试数据污染了生产环境的裁决记录

**影响**：历史裁决数据质量下降，影响精准度统计。

### 5.2 信号主题降级提示

**现象**：pro/limited账号在工作台看到"信号主题未开放，当前已降级为市场结论 + 候选漏斗判断"。

**问题分析**：
- 信号主题功能对非admin账号未开放
- 但降级提示中的"候选漏斗"对limited账号也是不可用的（见权限表）

**影响**：降级链路不完整，limited用户实际只能看到市场结论。

## 六、业务逻辑问题清单

### 6.1 三层股池数据不一致（严重）

**现象**：
- 系统机会池：203信号（信号总览确认）
- 用户关注池：0候选（候选漏斗确认）
- 持仓池：0持仓（持仓看板确认）

**问题分析**：
- 信号总览有203条信号，但候选漏斗为0，说明"信号→候选"的流转链路未打通
- 评分总览有8只自动短名单股票，但候选漏斗仍为0，说明"评分→候选"的流转也未打通

**影响**：系统机会池的数据无法流转到用户关注池，候选漏斗失去实际作用。

### 6.2 决策闭环不完整（严重）

**现象**：决策链路阶段显示"当前阶段：判断"，但后续"动作建议→风险披露→输出"阶段无数据。

**问题分析**：
- 决策链路只有"输入→取证→归因→判断"有数据
- "动作建议→风险披露→输出"阶段为空
- 计划单、持仓看板也为空

**影响**：从"判断"到"执行"的闭环断裂，系统只能辅助研究，无法支撑完整交易决策。

## 八、功能缺失清单

### 8.1 交易执行功能缺失（严重）

**缺失内容**：
- 无法从候选/信号直接生成交易订单
- 无法记录实际成交价格/数量
- 无法跟踪持仓成本/市值/浮动盈亏
- 无法计算组合收益率/风险指标

**影响**：系统停留在"研究"层面，无法形成"研究→决策→执行→跟踪"的完整闭环。

### 8.2 信号→候选自动流转缺失（严重）

**缺失内容**：
- 信号总览的203条信号无法自动/手动流转到候选漏斗
- 评分总览的8只短名单股票无法流转到候选漏斗
- 缺乏"加入关注池"的操作按钮

**影响**：研究员需要在系统外手动维护关注列表，增加了操作成本。

### 8.3 复盘与绩效分析缺失（中等）

**缺失内容**：
- 执行复盘只有简单的表单（订单ID、评级、滑点、笔记）
- 缺乏投资组合收益率曲线
- 缺乏策略回测功能
- 缺乏多维度绩效归因（行业、因子、时间等）

**影响**：无法评估研究质量和决策效果，难以形成持续改进闭环。

### 8.4 通知与告警功能不完善（中等）

**缺失内容**：
- 虽然任务调度中心显示50条未恢复告警，但前端没有告警通知中心
- 没有站内消息/邮件/短信通知机制
- 信号触发、任务完成等事件无实时推送

## 十、总结

### 10.1 系统整体评价

**优势**：
1. 功能模块丰富，覆盖了投研流程的主要环节
2. 数据量大（5509只股票、75087条新闻、203条信号）
3. 权限控制完善，三级角色分级清晰
4. 后台管理功能完善（任务调度、LLM管理、用户管理）
5. 前端交互流畅，筛选/搜索/分页功能正常

**核心问题**：
1. **数据流转断裂**：信号/评分数据无法流转到候选漏斗，候选无法流转到持仓
2. **执行链路缺失**：从"研究"到"交易执行"的闭环未完成
3. **监控数据失真**：数据源监控页面所有指标为0，Worker离线
4. **决策质量不可评估**：裁决精准度为0%，市场结论可信度为0%

### 10.2 优先级建议

| 优先级 | 问题 | 建议 |
|--------|------|------|
| P0 | 信号→候选流转缺失 | 增加"加入关注池"按钮，打通数据链路 |
| P0 | 交易执行功能缺失 | 增加模拟交易功能（下单→成交→持仓→复盘） |
| P0 | 监控数据全零 | 修复监控采集链路，确保Worker在线 |
| P1 | 决策精准度为0% | 修复统计逻辑或增加数据关联 |
| P1 | 市场结论可信度为0% | 完善结论生成逻辑 |
| P1 | 重复裁决记录 | 增加去重机制，清理测试数据 |
| P2 | 实时连接不稳定 | 优化WebSocket连接稳定性 |
| P2 | 新闻页面功能重叠 | 考虑合并国际/国内资讯为统一入口 |
