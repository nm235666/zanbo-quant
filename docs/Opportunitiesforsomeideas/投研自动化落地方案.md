# 投研自动化落地方案

> 目标：把“宏观 -> 行业 -> 个股 -> 策略实验 -> 报告输出”做成一条可重复、可验证、可迭代的主链路。

## 1. 我们要解决什么

当前的核心问题不是“缺功能”，而是“主观投研流程还没有被产品化”：

- 宏观判断靠人脑记忆和零散阅读
- 行业筛选靠经验判断，难以稳定复用
- 个股筛选标准不统一，难以自动排序
- 策略实验缺少统一的筛选和淘汰规则
- 报告、回测、候选池之间还没有形成闭环

因此，本方案的终局不是“让 AI 直接赚钱”，而是先把投研链路变成：

1. 系统自动收集和打分
2. 系统自动生成短名单
3. 系统自动批量实验策略
4. 系统自动沉淀为报告和可追踪结果

## 2. 现有基础

当前项目已经具备较好的底座，可以直接复用：

- 宏观数据看板：`/macro`
- 投研报告页：`/research/reports`
- 多角色公司分析：`/research/multi-role`
- 因子挖掘与回测：`/research/quant-factors`
- 投资信号系统：`/signals/*`
- 股票、财务、新闻、评分等基础数据链路

这意味着我们现在不需要从零搭一个“研究系统”，更适合在现有主链路上补齐“评分、筛选、实验、输出”。

## 3. 终局目标

### 3.1 产品目标

做一个能帮助你“少看、少想、少漏”的投研工作台：

- 每天自动告诉你宏观环境是否适合出手
- 每天自动告诉你哪些行业值得关注
- 每天自动告诉你哪些股票进入候选池
- 每天自动生成可回看的研究报告
- 每周自动批量实验策略，留下可复盘结果

### 3.2 验收标准

当方案进入稳定状态后，应满足：

- 你不用手工翻很多页面，也能拿到当天的关注清单
- 每一个候选标的都能追溯入选理由
- 每一个策略实验都有统一的分数和淘汰原因
- 回测结果、报告结果、信号结果可以互相跳转
- 前端展示稳定，空态和报错态可用

## 4. 第一批可落地方案

这一批只做最关键的一条链路：

### 4.1 宏观-行业-个股分层评分卡

把你现在的投研逻辑拆成三层分数：

- 宏观层：流动性、利率、汇率、风险偏好
- 行业层：景气度、估值位置、资金流、相对强弱
- 个股层：盈利质量、现金流、ROE、负债率、价格趋势

输出形式：

- 一个总分
- 三层分数
- 入选理由
- 风险提示

### 4.2 自动短名单

系统每天给出一个短名单，而不是全市场列表：

- 推荐关注行业
- 推荐关注股票
- 每只票的核心理由
- 对应的宏观背景和行业背景

### 4.3 策略处刑架

先不要急着“自动造策略”，先建立统一淘汰标准：

- 交易次数太少，淘汰
- 过拟合明显，淘汰
- 最大回撤过高，淘汰
- 测试集表现明显劣化，淘汰

这一步是策略实验能不能长期跑下去的关键。

## 5. 第二批方案

在第一批稳定后，再扩展到：

### 5.1 批量策略实验

对已有策略模板做参数搜索和组合搜索：

- 参数网格搜索
- 随机搜索
- Walk-forward 验证
- 结果自动排序

### 5.2 策略生成器

不是让模型“自由发挥”，而是限制在可控模板内生成：

- 因子组合
- 条件表达式
- 参数范围
- 风险约束

### 5.3 周度复盘报告

自动输出：

- 本周最强行业
- 本周最强短名单
- 本周策略实验表现
- 本周被淘汰策略及原因

## 6. 建议实施顺序

### 阶段 1：先把“看板”做出来

优先交付：

- 宏观评分
- 行业评分
- 个股评分
- 短名单页面

目标：

- 让你先看到系统是怎么筛选的
- 让筛选结果可解释、可对比

### 阶段 2：再把“回测筛选”做出来

优先交付：

- 策略实验任务
- 统一结果列表
- 淘汰原因展示
- 失败态和空态

目标：

- 让策略实验变成标准流程
- 让垃圾策略自动出局

### 阶段 3：最后再做“自动生成”

优先交付：

- 模板化策略生成
- 参数受控
- 输出可解释

目标：

- 降低人工构思成本
- 但不放开模型自由乱写

## 7. 当前可直接复用的页面和能力

- 宏观页：`/macro`
- 报告页：`/research/reports`
- 因子挖掘页：`/research/quant-factors`
- 多角色分析页：`/research/multi-role`
- 投资信号页：`/signals/overview`
- 主题页：`/signals/themes`
- 时间线页：`/signals/timeline`

这些页面可以作为第一批联动展示的承载位，不需要重新造一套前端。

## 8. 风险与边界

### 8.1 风险

- 自动生成策略很容易过拟合
- 单次回测结果不一定能代表真实可交易性
- 数据质量会直接影响评分和短名单
- 如果目标不收敛，系统会继续膨胀

### 8.2 边界

- 先做“辅助投研”，不要承诺“自动赚钱”
- 先做“可解释筛选”，再做“自动生成”
- 先做“可验证结果”，再做“更复杂模型”

## 9. 第一批建议的验收口径

如果我们开始做第一批，建议按以下标准验收：

- 页面可以展示
- 数据可以加载
- 空态和报错态可用
- 筛选结果能追溯理由
- 短名单和报告能互相跳转
- 回测任务和结果能稳定查看

## 10. 一句话结论

当前项目已经具备“研究系统底座”，最适合先落的是：

**宏观-行业-个股评分卡 + 自动短名单 + 策略处刑架。**

这三件事能最快把你的主观投研流程产品化，也最符合现在前端展示业务和测试可用性的优先级。
