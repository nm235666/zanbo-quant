## 4. 风险控制：防止"看起来很美的灾难"

你这种基本面+宏观的模型，最大的风险是**"价值陷阱"**（买了便宜但继续跌）和**"宏观钝化"**（信号太滞后）。

### 动态再平衡机制（Rebalance）

不要固定时间调仓，用**"事件触发制"**：

- **战略层触发**：当VIX单日暴涨>30%时，无视其他信号，仓位自动降到30%
- **战术层触发**：当持仓行业跌出前5名（行业动量衰竭），立即切换到新前2名，不等到月末
- **个股层触发**：
  - 止损：买入后回撤>15%（硬止损，不认死理）
  - 止盈：浮盈>30%后，回撤10%即止盈（移动止盈）
  - 基本面恶化：季报ROE环比下降>5%，立即剔除

### 压力测试（你必须做的）

用2015年股灾、2018年毛衣战、2022年俄乌冲突三段历史数据回测：
- 你的模型在那期间最大回撤是多少？（如果>40%，策略有毒）
- 是否连续3个月跑输现金？（策略失效期太长）

## 3. 自动化扫描框架："卧龙猎手"策略

不要试图复制"买卧龙电驱"这个具体动作，而是复制**"困境反转+产业链前瞻+龙头错杀"**这个模式。

### 策略逻辑（Python伪代码）

```python
class WOLongHunter:
    def __init__(self):
        self.concept_mapping = {
            "人形机器人": ["电机", "减速器", "传感器", "丝杠"],
            "AI算力": ["光模块", "液冷", "PCB", "变压器"],
            # 需要定期维护这个映射表
        }
    
    def macro_filter(self, date):
        """宏观层：全球热，A股冷"""
        sp500_return = get_index_return("SPX", 252, date)  # 近1年
        csi300_return = get_index_return("000300.SH", 252, date)
        
        # 全球涨了（标普>20%），中国没涨（沪深<10%）
        if sp500_return > 0.20 and csi300_return < 0.10:
            return True, f"中美温差{sp500_return-csi300_return:.1%}"
        return False, None
    
    def market_sentiment_filter(self, date):
        """市场温度：地量见地价"""
        turnover_pct = get_turnover_percentile("000300.SH", 750, date)  # 近3年分位
        volume_ma20 = get_volume_ma("000300.SH", 20, date)
        volume_pct = get_volume_percentile("000300.SH", 750, date)
        
        # 换手率<10%分位 且 成交额<20%分位 = 绝对冰点
        if turnover_pct < 0.10 and volume_pct < 0.20:
            return True, f"地量信号(换手{turnover_pct:.1%},成交{volume_pct:.1%})"
        return False, None
    
    def industry_chain_scan(self, hot_concept, date):
        """产业链扫描：从概念挖上游"""
        upstream_sectors = self.concept_mapping.get(hot_concept, [])
        
        candidates = []
        for sector in upstream_sectors:
            # 找出该细分领域营收前10的公司
            leaders = get_sector_leaders(sector, top_n=10, date=date)
            
            for stock in leaders:
                # 业务纯度：该业务占比>50%
                revenue_mix = get_business_mix(stock, date)
                if revenue_mix.get(sector, 0) > 0.5:
                    # 估值检查：PEG<1.5
                    peg = calculate_peg(stock, date)
                    if peg < 1.5:
                        candidates.append({
                            'code': stock,
                            'sector': sector,
                            'peg': peg,
                            'revenue_mix': revenue_mix[sector],
                            'reason': f"{hot_concept}上游{sector}龙头"
                        })
        return candidates
    
    def financial_quality_check(self, stock, date):
        """微观层：财报质量"""
        # 现金流质量
        cf_ratio = get_operating_cf(stock, date) / get_net_profit(stock, date)
        if cf_ratio < 1.0:  # 经营现金流<净利润，剔除
            return False
        
        # ROE稳定性
        roe_list = [get_roe(stock, date - timedelta(days=365*i)) for i in range(3)]
        if min(roe_list) < 0.10:  # 近3年ROE都>10%
            return False
        
        # 负债安全
        debt_ratio = get_debt_to_asset(stock, date)
        if debt_ratio > 0.50:
            return False
            
        return True
    
    def generate_signals(self, date):
        """主流程"""
        signals = []
        
        # Step 1: 宏观判断
        macro_ok, macro_msg = self.macro_filter(date)
        if not macro_ok:
            return []  # 宏观不对，空仓
        
        # Step 2: 市场情绪
        sentiment_ok, sentiment_msg = self.market_sentiment_filter(date)
        if not sentiment_ok:
            return []  # 不是冰点，不抄底
        
        # Step 3: 找热门概念（这里需要人工输入或另类数据）
        hot_concepts = detect_emerging_concepts(date)  # 通过新闻/融资/专利数据抓取
        
        for concept in hot_concepts:
            # Step 4: 产业链扫描
            candidates = self.industry_chain_scan(concept, date)
            
            for candidate in candidates:
                # Step 5: 财务质检
                if self.financial_quality_check(candidate['code'], date):
                    signals.append({
                        'code': candidate['code'],
                        'strategy': 'WOLong_V2',
                        'weight': 0.20,  # 单票20%上限
                        'holding_days': 180,  # 预期持有半年
                        'stop_loss': -0.15,  # 硬止损15%
                        'trailing_stop': 0.10,  # 浮盈回撤10%止盈
                        'reason': f"{macro_msg}|{sentiment_msg}|{candidate['reason']}"
                    })
        
        return signals
```

## 4. 你必须面对的两个致命问题

### 问题A：产业链映射是主观的
你当年能想到"机器人→电机"，是因为你有**产业认知**。机器不会自动知道"人形机器人需要无框力矩电机"，除非你把这层知识**硬编码**进去。

**解决方案**：
- 建立**"概念-产业链-个股"知识图谱**，定期手动维护（比如每季度更新一次映射表）
- 或者使用**供应链数据**（上市公司披露的"主要客户"信息），反向映射（谁给特斯拉/优必选供货）

### 问题B：左侧抄底需要极强的止损纪律
你拿住了赚了100%，但如果卧龙电驱跌了30%呢？你的逻辑（宏观好+行业好+公司好）**在账面上浮亏30%时依然成立**，这会让你陷入**价值陷阱**。

**必须添加的机械规则**：
```python
# 无论基本面多好，硬止损
if current_loss > 0.15:

# 浮盈保护
if current_profit > 0.30 and pullback_from_high > 0.10:
    force_close("浮盈30%后回撤10%，趋势结束")
```

## 1. 裸K位置的量化近似

你肉眼看到的"相对低位"，在代码里可以拆解为：

```python
def is_relative_low(stock, date):
    """判断当前是否处于'相对低位'（适合建底仓）"""
    
    # 维度1：趋势位置（周线级别）
    week_20 = get_weekly_ma(stock, 20, date)
    week_60 = get_weekly_ma(stock, 60, date)
    
    # 价格在长期均线附近，且偏离度不大（不是追高）
    price = get_price(stock, date)
    deviation_from_long = (price - week_60) / week_60
    
    # 条件：在长期均线上方（趋势仍在），但距离<15%（不是高位）
    if 0 < deviation_from_long < 0.15:
        trend_ok = True
    else:
        return False
    
    # 维度2：近期回调深度（日线级别）
    high_60 = get_high(stock, 60, date)  # 近60日高点
    pullback = (high_60 - price) / high_60
    
    # 条件：从近期高点回调>15%（给了安全边际），但<40%（不是崩盘）
    if 0.15 < pullback < 0.40:
        pullback_ok = True
    else:
        return False
    
    # 维度3：K线结构（简化版裸K）
    recent_candles = get_last_n_candles(stock, 10, date)
    
    # 检查是否出现"缩量十字星"或"长下影线"（止跌信号）
    last_candle = recent_candles[-1]
    body_ratio = abs(last_candle['close'] - last_candle['open']) / last_candle['high-low']
    lower_shadow = (min(last_candle['close'], last_candle['open']) - last_candle['low']) / last_candle['high-low']
    
    # 下影线>实体2倍，且量能<5日均量80%（缩量止跌）
    if lower_shadow > 2 * body_ratio and volume < ma_volume_5 * 0.8:
        structure_ok = True
    else:
        # 如果结构不满足，降低仓位或等待
        return False
    
    return True, {
        'distance_from_long_trend': deviation_from_long,
        'pullback_from_high': pullback,
        'signal': '缩量长下影，趋势中继回调位'
    }
```

**这只能近似你的"感觉"**，捕捉那种"跌不动了，但还没V型反转"的微妙状态。

## 2. 浮动仓位的自动化T0/T+波段

对于那1-2成浮动仓位，可以设定**网格交易**或**通道突破**：

```python
def floating_position_logic(stock, current_position_ratio, base_cost):
    """浮动仓位管理"""
    
    price = get_current_price(stock)
    profit_ratio = (price - base_cost) / base_cost
    
    # 场景A：底仓盈利>20%，启动浮动仓做T
    if profit_ratio > 0.20 and current_position_ratio < 0.50:  # 总仓位<50%
        # 日内网格：每涨3%卖1成，每跌3%买1成（基于底仓成本）
        grid_upper = base_cost * 1.20
        grid_lower = base_cost * 1.10
        
        if price > grid_upper * 1.03:
            return 'SELL', 0.10, '浮盈减仓'
        elif price < grid_upper * 0.97 and price > grid_lower:
            return 'BUY', 0.10, '回调加仓'
    
    # 场景B：跌破底仓成本8%，浮动仓止损（保护本金）
    if profit_ratio < -0.08 and current_position_ratio > 0.30:
        return 'SELL', current_position_ratio - 0.30, '跌破成本，浮动仓撤退'
    
    return 'HOLD', 0, None
```

## 3. "黑天鹅"的量化定义（最难的部分）

你提到的**"市场出现黑天鹅时打入5成仓位"**，这是典型的**左侧抄底**。

但**危险在于**：你以为的黑天鹅（黄金坑）可能是**飞刀接刀**（熊市才刚刚开始）。

必须设定**硬性的市场级信号**，而非个股信号：

```python
def black_swan_signal(date):
    """检测系统性黑天鹅，触发预备仓位"""
    
    signals = []
    score = 0
    
    # 信号1：全市场恐慌（全A股换手率<1%且千股跌停）
    market_turnover = get_all_stock_avg_turnover(date)
    limit_down_count = get_limit_down_count(date)
    if market_turnover < 0.01 and limit_down_count > 500:
        signals.append('千股跌停地量')
        score += 40
    
    # 信号2：波动率飙升（VIX-like指标）
    atr_20 = get_market_atr(20, date)
    atr_spike = atr_20 / get_market_atr(60, date)  # 短期ATR/长期ATR
    if atr_spike > 2.0:  # 波动率突然翻倍
        signals.append('波动率异常')
        score += 30
    
    # 信号3：流动性危机（国债收益率陡降+股指期货深度贴水）
    bond_yield_change = get_10y_bond_yield_change(5, date)  # 5日变化
    if bond_yield_change < -0.3:  # 避险资金涌入债市
        futures_premium = get_ih_premium(date)  # 上证50期指贴水
        if futures_premium < -0.02:  # 贴水>2%
            signals.append('流动性危机')
            score += 30
    
    # 只有当score>=60且你主观确认时，才允许动用预备仓位
    return score >= 60, signals, score
```

**关键纪律**：
- 黑天鹅仓位必须**分3批打入**（每次15-20%，间隔至少3天），防止接飞刀
- **止损线设在整体账户-10%**（一旦错判，立即撤退，保留火种）

## 5. 风险警告：这种仓位管理的"死亡陷阱"

你这套系统最大的隐患是**"预备仓位的诱惑"**：

- **场景1**：你看好一只票，建了3成底仓，小涨。你手痒，觉得"反正还有5成预备"，于是提前把预备仓加进去，变成重仓。然后回调，你割肉，实际亏损放大。
- **场景2**：遇到"假黑天鹅"（2024年初那种阴跌），你分3批打入预备仓，以为抄到底，结果市场继续跌，你满仓套牢，预备仓失去"预备"意义。

**必须设定的铁律**：
1. **预备仓位只有在"全市场级恐慌"时才解锁**，个股大跌（哪怕跌50%）也不触发
2. **浮动仓位如果连续3次做T失败（买卖点反了），自动冻结**，回归纯持有（防止手贱）
3. **裸K信号必须经过"次日确认"**：今天看到K线好，明天开盘前确认未破坏结构，才允许加仓（防止假突破）

你现在最需要的是**把"裸K感觉"转化为 checklist**，哪怕只有5条：

比如你的"低位" checklist：
- [ ] 周线仍在MA60之上（趋势未坏）
- [ ] 日线回调>15%但<40%（有空间但非崩盘）

**只有当机器检测到满足前3条时，才推送给你做后2条的人工确认**。

## 1. 量化定义："3天确认+右侧反转"模型

### 关键参数设定（基于你的描述）

| 概念 | 量化定义 | 参数建议 | 说明 |
|-----|---------|---------|------|
| **新低基准** | 近60日最低价（中期底部） | 60个交易日 | 太短假信号多，太长机会少 |
| **3天确认** | 连续3日收盘价 > 新低×1.005 | 缓冲0.5% | 允许微小下影线刺破但收盘收回 |
| **反转信号** | 放量阳线突破5日均线 | 量>5日均量120% | 确认资金回流，不是超跌反弹 |
| **分3次打入** | 金字塔加仓（1:2:3） | 8%→17%→25% | 首批试探，确认后加重 |
| **阶梯止损** | 跌破结构低点-3%启动 | 3%缓冲 | 防止正常波动触发止损 |

## 2. 核心代码框架（可直接接入Zanbo Quant）

```python
class RightSidePyramidStrategy:
    def __init__(self):
        self.params = {
            'lookback_low': 60,           # 找60日新低
            'confirmation_days': 3,       # 3天确认
            'volume_threshold': 1.2,        # 放量20%
            'buffer_breakdown': 0.03,       # 跌破新低3%止损
            'batches': [0.08, 0.17, 0.25]   # 三次加仓比例（累计50%）
        }
    
    def detect_structural_bottom(self, stock, date):
        """步骤1：结构确认 - 3天不破新低"""
        # 获取60日低点及日期
        low_60 = get_low(stock, self.params['lookback_low'], date)
        lowest_date = get_lowest_date(stock, self.params['lookback_low'], date)
        
        # 确保新低至少在3天前（给确认时间）
        days_since_low = trading_days_between(lowest_date, date)
        if days_since_low < self.params['confirmation_days']:
            return False, "新低出现不足3天"
        
        # 检查近3个交易日收盘价是否都站稳在新低上方
        recent_3 = get_last_n_bars(stock, 3, date)
        for bar in recent_3:
            # 收盘必须高于新低，且最好有逐步抬升
            if bar['close'] < low_60 * (1 + 0.005):  # 允许0.5%误差
                return False, f"近3日有收盘未站稳（{bar['date']}）"
        
        return True, {
            'structural_low': low_60,
            'confirmation_date': recent_3[0]['date'],  # 确认起点
            'strength': (recent_3[-1]['close'] - low_60) / low_60  # 反弹力度
        }
    
    def detect_reversal_confirmation(self, stock, date, structural_info):
        """步骤2：右侧反转 - 放量阳线"""
        today = get_daily_data(stock, date)
        yesterday = get_daily_data(stock, date, offset=1)
        ma5 = get_ma(stock, 5, date)
        vol_ma5 = get_volume_ma(stock, 5, date)
        
        # 条件1：站上5日线（趋势转折）
        if today['close'] < ma5 * 1.01:  # 1%缓冲
            return False, "未站上MA5"
        
        # 条件2：阳线实体（收盘>开盘，且实体明显）
        body = today['close'] - today['open']
        if body <= 0:
            return False, "非阳线"
        
        # 条件3：放量（资金确认）
        if today['volume'] < vol_ma5 * self.params['volume_threshold']:
            return False, "未放量"
        
        # 条件4：动能充足（阳线实体大于昨日振幅的60%，不是小阳）
        yest_range = yesterday['high'] - yesterday['low']
        if body < yest_range * 0.6:
            return False, "阳线力度不足"
        
        # 计算此次买入的止损位（结构低点下方3%）
        stop_loss = structural_info['structural_low'] * (1 - self.params['buffer_breakdown'])
        
        return True, {
            'entry_price': today['close'],
            'stop_loss': stop_loss,
            'volume_confirm': today['volume'] / vol_ma5,
            'breakthrough': "放量突破MA5"
        }
    
    def entry_batch(self, stock, date, current_reserve_used):
        """步骤3：分批建仓逻辑"""
        # 检查结构
        struct_ok, struct_info = self.detect_structural_bottom(stock, date)
        if not struct_ok:
            return None
        
        # 检查反转
        rev_ok, rev_info = self.detect_reversal_confirmation(stock, date, struct_info)
        if not rev_ok:
            return None
        
        # 确定当前是第几批
        batches_used = len(current_reserve_used)  # 已建仓次数
        if batches_used >= len(self.params['batches']):
            return None  # 3批已满
        
        batch_ratio = self.params['batches'][batches_used]
        
        return {
            'action': 'BUY_RESERVE',
            'ratio': batch_ratio,
            'price': rev_info['entry_price'],
            'stop_loss': rev_info['stop_loss'],  # 此批专属止损线
            'batch_no': batches_used + 1,
            'structural_low': struct_info['structural_low'],  # 记录用于后续比较
            'reason': f"右侧第{batches_used+1}批：3日不破{struct_info['structural_low']:.2f}+{rev_info['breakthrough']}"
        }
    
    def exit_logic(self, stock, date, positions, entry_history):
        """阶梯止损：跌破结构低点时，先平最早那批"""
        current_price = get_price(stock, date)
        
        # 只检查预备仓位（底仓单独管理）
        reserve_entries = [e for e in entry_history if e['type'] == 'reserve']
        if not reserve_entries:
            return None
        
        # 找到当前有效的结构低点（最后一批的structural_low或统一基准）
        # 策略：只要跌破任何一批买入时的"结构低点-3%"，启动减仓
        active_low = min([e['structural_low'] for e in reserve_entries])
        stop_line = active_low * (1 - self.params['buffer_breakdown'])
        
        if current_price < stop_line:
            # 逐步止损：平掉最早进入的那一批（FIFO）
            oldest_batch = reserve_entries[0]
            
            return {
                'action': 'SELL_RESERVE_FIFO',
                'ratio': oldest_batch['ratio'],
                'price': current_price,
                'reason': f"跌破结构止损线{stop_line:.2f}，平第1批预备仓",
                'remaining_reserve': sum([e['ratio'] for e in reserve_entries[1:]])
            }
        
        # 底仓移动止盈（独立逻辑）
        base_entry = next((e for e in entry_history if e['type'] == 'base'), None)
        if base_entry and current_price > base_entry['price'] * 1.30:
            # 浮盈30%后，回撤10%止盈底仓
            high_since_entry = get_high_since(stock, base_entry['date'], date)
            if current_price < high_since_entry * 0.90:
                return {
                    'action': 'SELL_BASE_TRAILING',
                    'ratio': 0.30,
                    'reason': "底仓浮盈30%后回撤10%，移动止盈"
                }
        
        return None
```

## 3. 关键风险：这种策略最怕什么

### 风险1："W底"变"M头"（最常见死亡模式）
你以为3天不破新低是底部确认，结果只是下跌中继：
```
Day 1: 大跌创新低（10元）→ 你以为到底
Day 2-4: 反弹3天（10.5元）→ 触发你的买入信号
Day 5: 再次暴跌破新低（9.5元）→ 你刚买就套，止损在最低
```

**对策**：
- **批次间隔**：不要连续3天买，而是**第1天买8%，观察5天，如果浮盈>5%再买第2批17%**，否则停止加仓
- **量能过滤**：买入当天不仅要放量，还要是**近10日最大量**（确认资金真实流入，不是缩量反弹）

### 风险2：分3天买入的"时间错配"
你说"分3次，3天"，但如果：
- 第1天买入后，第2天直接跌破新低
- 你止损第1批，但第3天又涨回去

**对策**：
```python
# 修改分批逻辑：不是时间分散，而是价格梯度
batch_prices = [
    {'ratio': 0.08, 'trigger': 'initial_break'},      # 首次突破
    {'ratio': 0.17, 'trigger': 'profit_5pct'},      # 首批浮盈5%后
    {'ratio': 0.25, 'trigger': 'trend_confirm_ma20'} # 站稳MA20后
]
```
只有前一匹盈利，才放下一匹进场，**避免在下跌通道中持续加仓**。

### 风险3：黑天鹅与你的5成仓位冲突
你说市场黑天鹅时打5成，但右侧策略要求**趋势确认后才买**。真黑天鹅时（千股跌停），右侧策略会**空仓观望**，不会发出买入信号。

**矛盾解决**：
- **分层策略**：右侧策略管那3成底仓+2成浮动（常规操作）
- **黑天鹅策略**（左侧抄底）独立决策，只在VIX>40或全市场PE<历史10%分位时启动，且必须**分10天定投式买入**，不是一次性5成

## 4. 给你 tonight 就能验证的工具

### 历史回测检查清单（针对卧龙电驱案例）
打开你的交易软件，检查如果严格执行这套规则：
1. 2023年你买入卧龙电驱的那段时间，**60日低点出现在哪一天**？
2. 你买入当天，是否满足"突破5日线+放量20%"？
3. 如果你分3批（8%-17%-25%）买入，后两批的买入价是更高还是更低？

## 1. 支撑带与阻力带的量化定义

不要用固定价格，要用**"流动区间"**（基于近期筹码密集区+波动率）：

```python
class DynamicBands:
    def __init__(self, lookback=20, atr_multiplier=1.5):
        self.lookback = lookback      # 观察近期20日
        self.atr_mul = atr_multiplier  # ATR缓冲系数
    
    def calculate_bands(self, stock, date):
        # 获取近期K线
        df = get_last_n_bars(stock, self.lookback, date)
        
        # 方法1：基于成交量加权平均价（VWAP）± ATR
        vwap = (df['close'] * df['volume']).sum() / df['volume'].sum()
        atr = calculate_atr(stock, 14, date)  # 14日ATR
        
        # 支撑带 = VWAP - 1.5×ATR 至 VWAP - 0.5×ATR（区间低吸）
        support_zone = (vwap - self.atr_mul * atr, vwap - 0.5 * atr)
        
        # 阻力带 = VWAP + 0.5×ATR 至 VWAP + 1.5×ATR（区间高抛）
        resistance_zone = (vwap + 0.5 * atr, vwap + self.atr_mul * atr)
        
        # 方法2：基于近期平台高低点（备用）
        recent_high = df['high'].max()
        recent_low = df['low'].min()
        # 如果价格在近期高低点中间40%区域，认定为震荡市，方法1有效
        # 如果价格突破近期高点的95%，转为趋势跟踪模式（阻力带失效）
        
        return {
            'support': support_zone,      # (下轨, 上轨) 低吸区
            'resistance': resistance_zone, # (下轨, 上轨) 高抛区
            'center': vwap,               # 中轴
            'trend_mode': self._detect_trend(df)  # 震荡/趋势判断
        }
    
    def _detect_trend(self, df):
        """判断当前是震荡还是趋势（决定支阻是否有效）"""
        # 用ADX指标或简单的高低点的位移判断
        price_range = df['high'].max() - df['low'].min()
        displacement = abs(df['close'].iloc[-1] - df['close'].iloc[0])
        
        # 如果价格位移 < 区间范围的30%，认定为震荡（支阻有效）
        if displacement < price_range * 0.3:
            return 'RANGE'
        else:
            return 'TREND'
```

## 2. 日内短T（T+0回转）策略

假设你有**底仓30%**（长期持有不动），日内短T是用这30%的筹码做**回转交易**（A股T+1规则下，先卖后买或先买后卖，但仓位不变）：

```python
class IntradayT0:
    def __init__(self, base_position=0.30):
        self.base = base_position  # 底仓比例，用来做T的筹码
    
    def scan_intraday(self, stock, current_data, bands):
        """
        当前数据包含：open, high, low, current_price, volume_ratio
        bands来自DynamicBands的计算
        """
        price = current_data['price']
        support = bands['support']      # (low, high)
        resistance = bands['resistance']  # (low, high)
        
        signals = []
        
        # 场景A：价格跌入支撑带（低吸机会）
        if support[0] <= price <= support[1]:
            # 额外确认：出现止跌K线（长下影或缩量）
            if self._is_bounce_signal(current_data):
                # 动作：用底仓做T，先买入（变相加仓），待会高位T出
                # 但A股T+1，所以实际是：如果昨日已买入今日可卖，或先卖后买
                signals.append({
                    'type': 'T0_BUY',
                    'ratio': 0.05,  # 动用5%的机动资金（来自之前的10-20%浮动）
                    'target_sell': resistance[0],  # 目标阻力带下轨
                    'stop_loss': support[0] * 0.98,  # 跌破支撑带下轨2%止损
                    'reason': '支撑带内出现止跌信号，日内低吸'
                })
        
        # 场景B：价格涨至阻力带（高抛机会）
        elif resistance[0] <= price <= resistance[1]:
            # 额外确认：出现滞涨信号（长上影或放量滞涨）
            if self._is_stall_signal(current_data):
                signals.append({
                    'type': 'T0_SELL',
                    'ratio': 0.05,  # 卖出5%（相当于做T降成本）
                    'target_buy_back': support[1],  # 跌回支撑带上轨买回
                    'urgent_buy_back': resistance[1] * 1.02,  # 突破追回来
                    'reason': '阻力带内滞涨，日内高抛'
                })
        
        return signals
    
    def _is_bounce_signal(self, data):
        """止跌信号：长下影线 + 缩量"""
        body = abs(data['close'] - data['open'])
        lower_shadow = min(data['open'], data['close']) - data['low']
        # 下影线是实体的2倍以上
        return lower_shadow > body * 2 and data['volume'] < data['vol_ma5'] * 0.9
    
    def _is_stall_signal(self, data):
        """滞涨信号：长上影线 + 放量但价格不涨"""
        body = abs(data['close'] - data['open'])
        upper_shadow = data['high'] - max(data['open'], data['close'])
        return upper_shadow > body * 1.5 and data['volume'] > data['vol_ma5'] * 1.3
```

## 3. 波段操作（Swing Trading）

波段操作用于那**10-20%的浮动仓位**，持仓周期3-10天：

```python
class SwingOperation:
    def __init__(self):
        self.swing_ratio = 0.15  # 浮动仓位15%
    
    def band_trading(self, stock, date, bands, current_position):
        price = get_price(stock, date)
        support = bands['support']
        resistance = bands['resistance']
        trend = bands['trend_mode']
        
        # 只在震荡市做波段（趋势市不做逆向波段，顺趋势持仓）
        if trend == 'TREND':
            return None
        
        # 当前已有浮动仓位
        current_float = current_position.get('float', 0)
        
        # 策略1：支撑带建仓
        if price <= support[1] and current_float < 0.05:  # 接近支撑且空仓
            return {
                'action': 'BAND_BUY',
                'ratio': 0.10,  # 建10%浮动仓
                'price_zone': support,
                'target': resistance[0],  # 目标阻力带下轨
                'stop': support[0] * 0.97,  # 跌破支撑下轨3%止损
                'holding_days': 5,  # 最多拿5天，不到目标也走
                'type': 'swing'
            }
        
        # 策略2：阻力带减仓
        elif price >= resistance[0] and current_float > 0.05:  # 接近阻力且有仓
            return {
                'action': 'BAND_SELL',
                'ratio': current_float,  # 全平浮动仓
                'price_zone': resistance,
                'reason': '触及阻力带，波段止盈',
                'type': 'swing'
            }
        
        # 策略3：时间止损（持仓超5天未达目标，无论盈亏离场）
        if current_position.get('float_days', 0) > 5:
            return {
                'action': 'BAND_EXIT_TIME',
                'ratio': current_float,
                'reason': '波段超时，流动性优先'
            }
        
        return None
```

## 5. 关键纪律与死亡陷阱

### 你必须画死的红线：

1. **支撑带≠无限抄底区**
   - 第一次触及支撑带：可以买（浮动仓）
   - **如果跌破支撑带下轨3%且收盘未收回**：无论浮动仓亏多少，**立即止损**
   - 第二次回测支撑带（已跌破后反弹上来）：**不买**（支撑变阻力）

2. **日内T的止损**
   - 日内低吸后如果继续跌，**当天收盘前必须T出**（或隔夜止损），不能变成"被动加仓"

3. **波段与趋势的区别**
   - 震荡市（ADX<20）：用浮动仓做波段（高抛低吸）
   - 趋势市（ADX>30）：浮动仓转为趋势加仓（只低吸，不高抛，让利润奔跑）

4. **阻力带的处理**
   - 触及阻力带：浮动仓必须减仓（不管后面涨不涨，先锁定波段利润）
   - **突破阻力带**：不是加仓信号，是**观望**（等回踩确认阻力变支撑）

## 1. 自动识别"有效"前高前低（Swing Points）

不是每个局部高低点都值得画线，需要**结构强度**过滤：

```python
class SwingDetector:
    def __init__(self, left_bars=5, right_bars=3, volume_confirm=True):
        """
        left_bars: 左侧需要多少根K线确认这是高点
        right_bars: 右侧需要多少根K线确认反转
        volume_confirm: 是否要求关键位伴随成交量异常（放量滞涨/止跌）
        """
        self.left = left_bars
        self.right = right_bars
        self.vol_confirm = volume_confirm
    
    def detect_swing_highs(self, df):
        """识别有效前高（阻力位候选）"""
        swing_highs = []
        
        for i in range(self.left, len(df) - self.right):
            current = df.iloc[i]
            
            # 条件1：当前是局部最高点（高于左右各N根K线）
            left_section = df.iloc[i-self.left:i]['high']
            right_section = df.iloc[i+1:i+1+self.right]['high']
            
            if current['high'] > left_section.max() and current['high'] > right_section.max():
                
                # 条件2：成交量确认（高点放量，说明在此处有大量换手）
                vol_avg = df.iloc[i-10:i]['volume'].mean()
                if self.vol_confirm and current['volume'] < vol_avg * 1.2:
                    continue  # 缩量高点，可能是假突破，忽略
                
                # 条件3：K线实体质量（长上影线更好，代表抛压）
                upper_shadow = current['high'] - max(current['open'], current['close'])
                body = abs(current['close'] - current['open'])
                if upper_shadow < body * 0.5:
                    continue  # 实体太大，上影不够长，可能是趋势延续而非反转点
                
                swing_highs.append({
                    'price': current['high'],
                    'date': current['date'],
                    'volume': current['volume'],
                    'strength': upper_shadow / (body + 0.001),  # 影线倍数
                    'touch_count': 1  # 后续会被更新
                })
        
        return self._merge_close_levels(swing_highs, tolerance=0.02)  # 合并2%内的相近高点
    
    def detect_swing_lows(self, df):
        """识别有效前低（支撑位候选）"""
        swing_lows = []
        
        for i in range(self.left, len(df) - self.right):
            current = df.iloc[i]
            
            left_section = df.iloc[i-self.left:i]['low']
            right_section = df.iloc[i+1:i+1+self.right]['low']
            
            if current['low'] < left_section.min() and current['low'] < right_section.min():
                
                # 成交量确认：低点缩量（止跌）或放量（恐慌抛售）
                vol_avg = df.iloc[i-10:i]['volume'].mean()
                if current['volume'] > vol_avg * 1.5:  # 放量长下影是更好信号
                    pass  # 放量恐慌，可能是底
                elif current['volume'] < vol_avg * 0.8:  # 缩量止跌
                    pass
                else:
                    continue
                
                lower_shadow = min(current['open'], current['close']) - current['low']
                body = abs(current['close'] - current['open'])
                
                swing_lows.append({
                    'price': current['low'],
                    'date': current['date'],
                    'volume': current['volume'],
                    'strength': lower_shadow / (body + 0.001),
                    'touch_count': 1
                })
        
        return self._merge_close_levels(swing_lows, tolerance=0.02)
    
    def _merge_close_levels(self, levels, tolerance=0.02):
        """合并接近的价格水平（形成阻力带/支撑带而不是单一线）"""
        if not levels:
            return []
        
        # 按价格排序
        levels.sort(key=lambda x: x['price'])
        merged = []
        
        current_zone = [levels[0]]
        for level in levels[1:]:
            if (level['price'] - current_zone[0]['price']) / current_zone[0]['price'] < tolerance:
                current_zone.append(level)
            else:
                # 计算该区域的统计特征
                avg_price = sum([x['price'] for x in current_zone]) / len(current_zone)
                total_volume = sum([x['volume'] for x in current_zone])
                max_strength = max([x['strength'] for x in current_zone])
                
                merged.append({
                    'price': avg_price,
                    'zone_low': min([x['price'] for x in current_zone]) * 0.995,
                    'zone_high': max([x['price'] for x in current_zone]) * 1.005,
                    'volume': total_volume,
                    'strength': max_strength,
                    'test_count': len(current_zone),
                    'type': 'resistance' if current_zone[0].get('type') != 'support' else 'support'
                })
                current_zone = [level]
        
        return merged
```

## 2. 构建多级支撑阻力带（Support/Resistance Zones）

你不是画一条线，而是画一个**"价格磁吸区"**（Zone），基于历史在该价位的反复测试：

```python
class ZoneBuilder:
    def __init__(self, df_daily):
        self.swings = SwingDetector()
        self.resistances = self.swings.detect_swing_highs(df_daily)  # 前高区
        self.supports = self.swings.detect_swing_lows(df_daily)     # 前低区
    
    def get_active_zones(self, current_price, lookback_days=60):
        """
        获取当前有效的支撑/阻力带
        只考虑近期（60日内）被测试过的高低区，太久远的可能失效
        """
        active_res = []
        active_sup = []
        
        # 阻力带：价格在当前价上方，且近期（20日内）有K线触及测试
        for res in self.resistances:
            if res['price'] > current_price * 1.01:  # 在当前价上方1%以上
                if self._is_tested_recently(res['zone_low'], res['zone_high'], lookback_days):
                    active_res.append(res)
        
        # 支撑带：价格在当前价下方
        for sup in self.supports:
            if sup['price'] < current_price * 0.99:
                if self._is_tested_recently(sup['zone_low'], sup['zone_high'], lookback_days):
                    active_sup.append(sup)
        
        # 按距离当前价格排序（最近的支撑/阻力）
        active_res.sort(key=lambda x: x['price'])
        active_sup.sort(key=lambda x: x['price'], reverse=True)
        
        return {
            'resistance_1': active_res[0] if len(active_res) > 0 else None,  # 第一阻力
            'resistance_2': active_res[1] if len(active_res) > 1 else None,  # 第二阻力（前高）
            'support_1': active_sup[0] if len(active_sup) > 0 else None,    # 第一支撑
            'support_2': active_sup[1] if len(active_sup) > 1 else None,    # 第二支撑（强支撑）
        }
    
    def _is_tested_recently(self, zone_low, zone_high, days):
        """检查该价格区近期是否被K线触及（影线或实体）"""
        recent_data = get_last_n_bars(days)  # 最近N日
        for _, row in recent_data.iterrows():
            # 影线触及或实体穿过
            if (row['low'] <= zone_high and row['high'] >= zone_low):
                return True
        return False
    
    def zone_quality_score(self, zone):
        """评估支撑/阻力带的强度"""
        score = 0
        # 测试次数越多越强
        score += min(zone.get('test_count', 1) * 10, 50)
        # 成交量越大（筹码交换充分），支撑阻力越强
        score += min(zone['volume'] / 1000000, 30)  # 假设百万手为单位
        # 影线强度（长影线代表强烈 rejection）
        score += zone.get('strength', 1) * 10
        return score
```

```python
                        'action': 'ZONE_BUY_S1',
                        'ratio': 0.10,

                        'hard_stop': zones['support_1']['zone_low'] * 0.98,  # 跌破带下轨止损

                    'action': 'INTRADAY_DIP',
                    'ratio': 0.05,

                    'target_sell': zones['resistance_1']['zone_low'] if zones['resistance_1'] else price * 1.03
                })
        
        # S2（第二支撑，通常是前低/强支撑）- 更激进，可用预备仓试探
        elif zones['support_2'] and zones['support_2']['zone_high'] >= price >= zones['support_2']['zone_low']:

                    'layer': 'RESERVE_BATCH_1',
                    'action': 'TEST_S2',
                    'ratio': 0.08,
                    'stop': zones['support_2']['zone_low'] * 0.97,
                    'reason': '第二支撑带(S2)附近3日确认，右侧试探'
                })
        
        # ========== 场景2：价格进入阻力带（R1）==========
        elif zones['resistance_1'] and zones['resistance_1']['zone_low'] <= price <= zones['resistance_1']['zone_high']:
            quality = self.zones.zone_quality_score(zones['resistance_1'])
            
            # 浮动仓必须减仓（波段止盈）
            if current_pos['float'] > 0:
                decisions.append({
                    'layer': 'FLOAT',
                    'action': 'ZONE_SELL_R1',
                    'ratio': current_pos['float'],  # 全平浮动
                    'reason': f'触及阻力带1(质量{quality})，波段离场',
                    'note': '若放量突破R1上轨，次日可追（趋势转右侧）'
                })
            
            # 日内T高抛机会
            if is_stall_candle(date):  # 滞涨信号
                decisions.append({
                    'layer': 'BASE_T0',
                    'action': 'INTRADAY_POP',
                    'ratio': 0.05,
                    'reason': '阻力带内滞涨，日内高抛'
                })
        
        # ========== 场景3：突破关键带（趋势转换）==========
        # 突破R1且收盘站稳R1上轨1%以上：阻力变支撑，启动右侧加仓（预备仓第2/3批）
        if price > zones['resistance_1']['zone_high'] * 1.01:
            decisions.append({
                'layer': 'RESERVE_BATCH_2',
                'action': 'BREAKOUT_ADD',
                'ratio': 0.17,
                'new_stop': zones['resistance_1']['zone_high'],  # 新止损设在原阻力（现支撑）
                'reason': '放量突破阻力带1，右侧确认，加仓'
            })
        
        # 跌破S1且收盘低于S1下轨3%以上：支撑变阻力，全部浮动仓止损，预备仓停止入场
        if price < zones['support_1']['zone_low'] * 0.97:
            decisions.insert(0, {  # 紧急置顶
                'layer': 'EMERGENCY',
                'action': 'ZONE_BREAKDOWN',
                'ratio': current_pos['float'] + current_pos['reserve_used'],
                'keep_base': 0.15,  # 只保留15%底仓，其余砍仓
                'reason': f'跌破支撑带1下轨3%，结构破坏，强制止损'
            })
        
        return decisions
```

## 5. 关键纪律（比之前更严格）

1. **带的宽度管理**
   - S1/R1的宽度通常设为**近期ATR的0.8-1.2倍**（如果自动合并的高低点评分太宽，人工收窄到2-3%区间）
   - **不操作带的中轨以外**：只在带下轨（支撑）或带上轨（阻力）附近操作，带内中轨是"真空区"，持仓但不增不减

2. **带的失效条件**
   - 支撑带被**实体（非影线）**收盘跌破3%以上，且**次日反抽站不回** → 该支撑带永久删除，转为阻力带
   - 阻力带被**放量阳线**突破且**回踩不破** → 转为新的支撑带（右侧加仓触发）

3. **多周期验证**
   - 日线S1如果和**周线MA20**或**月线前低**重合 → 质量分+30，可以加大仓位
   - 纯日线级别的S1 → 只用浮动仓，不动预备仓

4. **假突破过滤**
   - 日内突破R1上轨但**收盘回到带内** → 假突破，次日在R1上轨挂空单（对冲或减仓）

```python
    C = swing['C']['price']
```

### 风险1："三点结构"在趋势末期的失效
当股票处于**第三浪或第五浪**（艾略特波浪理论）时，会出现**"多头陷阱"**：
- 看起来有完美的A-B-C结构
- 你在C点买入（以为是新一波起点）
- 结果是**双顶M头**的开始，C点之后直接A杀跌破A点

**对策**：给"三点结构"加**生命周期限制**：
- 从A点到C点的**时间不能超过60个交易日**（太久的结构无效）
- 整个结构的**涨幅累计不能超过100%**（翻倍后的结构慎做）
- 必须配合**宏观层和行业层**的共振（卧龙电驱当时有机器人风口，现在可能没有了）

## 3. 实战决策树（整合版）

```python
def trading_decision(current_price, sr_zones, swing_structure, kline_pattern):
    """
    主系统：支撑阻力（sr_zones: S1, R1）
    辅助系统：三点结构（swing: A, B, C位置状态）
    """
    
    # ========== 第一层：生存检查（支撑带硬规则） ==========
    
    # 场景1：跌破支撑带（无论三点说什么）
    if current_price < sr_zones['S1_low'] * 0.97:
        return {
            'action': 'EMERGENCY_EXIT',
            'cut_ratio': 0.80,  # 砍掉浮动+预备，只留5%观察
            'reason': '支撑带S1有效跌破，三点结构暂时失效，保命优先'
        }
    
    # 场景2：触及支撑带（主系统触发）
    if sr_zones['S1_low'] <= current_price <= sr_zones['S1_high']:
        
        # 检查辅助系统：三点结构的位置
        distance_to_B = abs(current_price - swing_structure['B']) / swing_structure['B']
        
        # 冲突A：如果在B点附近（距离<5%），可能是双顶，不买
        if distance_to_B < 0.05:
            return {
                'action': 'WAIT_OR_LIGHT',
                'ratio': 0.02,  # 最多2%试探，不当成S1买点
                'reason': '支撑带S1触及，但三点显示接近前高B，防M头，轻仓试探'
            }
        
        # 冲突B：如果C点已形成且C≈S1（共振）
        if swing_structure['C_confirmed'] and \
           abs(swing_structure['C'] - current_price) / current_price < 0.03:
            
            if kline_pattern == 'bounce':  # 止跌K线
                return {
                    'action': 'FULL_BUILD',
                    'base': 0.30,      # 底仓
                    'float': 0.10,     # 浮动
                    'reserve_1': 0.08, # 预备第一批
                    'reason': '支撑带S1+三点C点共振，结构完美，重仓'
                }
        
        # 冲突C：如果三点显示还在B-C途中（C未确认）
        if not swing_structure['C_confirmed']:
            return {
                'action': 'SCALP_ONLY',
                'ratio': 0.05,  # 只做日内T，不过夜
                'reason': '支撑带触及，但三点结构C点未确认，可能是下跌中继，日内短T'
            }
    
    # 场景3：触及阻力带R1
    if sr_zones['R1_low'] <= current_price <= sr_zones['R1_high']:
        
        # 检查三点：如果正在形成新的A'-B'（突破后回踩）
        if swing_structure['new_A_formed'] and current_price > swing_structure['new_A']:
            return {
                'action': 'HOLD_TREND',  # 浮动仓不走，可能突破
                'trail_stop': sr_zones['R1_low']  # 把止损移到R1下轨
            }
        else:
            return {
                'action': 'PROFIT_TAKE',
                'ratio': 0.10,  # 平浮动仓
                'reason': '触及R1且三点未确认新趋势，波段止盈'
            }
    
    # ========== 第二层：趋势加速（三点主导） ==========
    
    # 如果价格已经突破R1且站稳（三点确认新A点）
    if current_price > sr_zones['R1_high'] * 1.02 and swing_structure['breakout_confirmed']:
        return {
            'action': 'PYRAMID_ADD',
            'reserve_batch': 2,  # 预备第2批17%
            'new_stop': sr_zones['R1_high'],  # 新止损变为原阻力
            'reason': '三点确认趋势延续，突破有效，右侧加仓'
        }
    
    return {'action': 'HOLD'}
```

## 1. 你描述的两种场景量化定义

### 场景A："上涨途中收口"（防御信号）
```python
def detect_upper_squeeze(df, band_period=20, squeeze_threshold=0.10):
    """
    上涨途中的收口（潜在见顶/中继）
    """
    current = df.iloc[-1]
    prev = df.iloc[-5]  # 5周期前
    
    # 条件1：价格在布林上轨附近运行（上涨趋势确认）
    upper_touch = current['close'] > current['BB_upper'] * 0.98
    
    # 条件2：带宽收窄（波动率下降）
    bandwidth_now = (current['BB_upper'] - current['BB_lower']) / current['BB_middle']
    bandwidth_before = (prev['BB_upper'] - prev['BB_lower']) / prev['BB_middle']
    
    squeeze = bandwidth_now < bandwidth_before * (1 - squeeze_threshold)
    
    # 条件3：收口发生在高位（区别于低位盘整）
    high_level = current['close'] > df['close'].quantile(0.70)  # 处于近期70%分位以上
    
    if upper_touch and squeeze and high_level:
        return {
            'signal': 'DEFENSE_SQUEEZE',
            'action': 'reduce_float',  # 减持浮动仓
            'reason': '上涨高位收口，波动率压缩，防回调',
            'bandwidth': bandwidth_now,
            'position_suggestion': 0.10  # 只保留10%浮动，其余止盈
        }
    return None
```

**你的逻辑**：波动率下降+高位 = 资金犹豫 = 可能变盘下跌 → **轻仓防御**

**危险**：这可能是**趋势加速前的最后盘整**（俗称"空中加油"），你减仓后错过主升浪。

### 场景B："低位横盘收口"（进攻信号）
```python
def detect_lower_squeeze(df, band_period=20, squeeze_threshold=0.15, consolidation_days=10):
    """
    低位收口（潜在爆发前兆）
    """
    recent = df.iloc[-consolidation_days:]
    current = df.iloc[-1]
    
    # 条件1：价格在布林中轨下方（低位）
    low_level = current['close'] < current['BB_middle'] * 1.02
    
    # 条件2：长期横盘（中轨走平）
    middle_slope = abs(recent['BB_middle'].iloc[-1] - recent['BB_middle'].iloc[0]) / recent['BB_middle'].iloc[0]
    flat = middle_slope < 0.03  # 3%以内波动视为横盘
    
    # 条件3：带宽极度收窄（历史低位10%分位）
    bandwidth = (current['BB_upper'] - current['BB_lower']) / current['BB_middle']
    bandwidth_hist = df['bandwidth'].quantile(0.10)  # 历史10%分位
    
    extreme_squeeze = bandwidth < bandwidth_hist
    
    if low_level and flat and extreme_squeeze:
        return {
            'signal': 'ATTACK_SQUEEZE',
            'action': 'build_position',  # 建仓/加仓
            'reason': '低位极端收口，变盘在即，方向选择上破概率大',
            'bandwidth': bandwidth,
            'entry_trigger': current['BB_upper'] * 1.01,  # 突破上轨进场
            'stop_loss': current['BB_lower'] * 0.99  # 下轨止损
        }
    return None
```

**你的逻辑**：低位+波动率压缩到极致 = 弹簧压到底 = 即将反弹 → **进场做多**

**危险**：这可能是**下跌中继的静默期**（俗称"下跌楔形整理"），你进场后遇到暴跌破位。

## 3. 你必须知道的布林带陷阱

### 陷阱1："收口"后的方向选择不是随机的
你以为低位收口"上下概率各50%"，但实际上：
- 如果收口发生在**下跌趋势的末端**（长跌之后），向下破位概率**60%**（恐慌性杀跌最后一脚）
- 如果收口发生在**上涨趋势的中继**（盘整后），向上突破概率**70%**

**对策**：看收口前的**趋势方向**（中轨斜率），不要只看收口本身。

### 陷阱2：周期错配（最致命）
- 你在**日线**上看到布林带收口（认为要变盘）
- 但在**周线**上，布林带刚刚开口向下（大级别下跌刚开始）
- **结果**：你在日线"低位"买入，结果是周线下跌的中继站

**对策**：必须**多周期验证**：
- 日线收口 + 周线中轨向上 = 高胜率做多
- 日线收口 + 周线中轨向下 = 不做多（或只做日内T）

### 陷阱3："假收口"（Fake Squeeze）
有时候收口后价格**轻微突破上轨**（你以为要涨了），然后**迅速回落到带内**并暴跌（多头陷阱）。

**对策**：加入**成交量确认**：
- 向上突破上轨时，成交量必须 > 5日均量130%
- 否则视为"假突破"，即使收口也不进场

```python
def trend_following_entry(monthly_trend, weekly_trend, daily_setup):
    """
    大周期向上时的日线追多（不是盲目追，是突破追）
    """
    if monthly_trend != 'UP' or weekly_trend != 'UP':
        return None  # 大周期不满足，不追
    
    # 即使大周期向上，也不追涨，只追"突破确认"
    if daily_setup['type'] == 'BREAKOUT_R1':
        # 条件1：突破必须是**放量**（>5日均量130%）
        if not daily_setup['volume_confirm']:
            return {'action': 'WAIT', 'reason': '无量突破，假突破概率高'}
        
        # 条件2：突破后**回踩不破**原R1（支撑变阻力验证）
        if not daily_setup['pullback_test']:
            return {'action': 'WAIT', 'reason': '未回踩确认，不追'}
        
        # 条件3：布林带必须**开口向上**（趋势加速，非收口）
        if daily_setup['bb_status'] != 'EXPANDING_UP':
            return {'action': 'LIGHT', 'ratio': 0.10, 'reason': '布林带未开口，可能是末端加速，轻仓'}
        
        return {
            'action': 'PYRAMID_ADD',  # 预备仓第2/3批
            'ratio': 0.17,  # 第2批
            'stop': daily_setup['r1_original'],  # 原阻力位变止损
            'reason': '大小周期共振，突破追多（高胜率）'
        }
    
    return None
```

```python
def counter_trend_caution(monthly_trend, weekly_trend, daily_setup):
    """
    大周期向下时，小周期的"谨慎"策略
    """
    if monthly_trend == 'UP' or weekly_trend == 'UP':

    # 大周期双向下
    if daily_setup['type'] == 'S1_TOUCH':
        # 只做**超短反弹**（日内T或1-2日持有），不建仓
        return {

            'ratio': 0.05,  # 最多5%
            'holding_days': 2,  # 2天内必须走
            'profit_target': daily_setup['resistance_1'] * 0.98,  # 不到前高就撤
            'stop_loss': daily_setup['support_1'] * 0.98,  # 严格止损
            'reason': '大周期向下，小周期反弹只做超短，不留隔夜重仓'

    if daily_setup['type'] == 'BREAKOUT':
```

## 2. 小周期右侧信号的精确标准（油门怎么踩）

既然大周期给了绿灯，小周期的"右侧"不再是简单的"突破"，而是**"突破确认"**——防止假突破。

### 定义：小周期（日线/4小时）的右侧买点

```python
def small_cycle_right_signal(daily_df, hourly_df, sr_zones):
    price = daily_df['close'].iloc[-1]
    
    # 条件1：结构突破（突破R1或前高，且不是假突破）
    breakout = price > sr_zones['R1_high'] * 1.01  # 站上阻力带1%以上
    
    # 条件2：波动率确认（布林带开口向上，拒绝收口）
    bb_upper = daily_df['BB_upper'].iloc[-1]
    bb_upper_prev = daily_df['BB_upper'].iloc[-5]
    expanding = bb_upper > bb_upper_prev * 1.02  # 上轨向上开口
    
    # 条件3：动量确认（不是无量空涨）
    volume = daily_df['volume'].iloc[-1] > daily_df['volume'].rolling(5).mean().iloc[-1] * 1.3
    macd_confirm = daily_df['macd'].iloc[-1] > daily_df['macd'].iloc[-5]  # MACD持续向上
    
    # 条件4：时间过滤（突破后回踩不破，或连续2根K线站稳）
    if breakout:
        # 检查过去3根K线是否都站稳突破位之上
        recent_closes = daily_df['close'].iloc[-3:]
        stable = all(c > sr_zones['R1_high'] for c in recent_closes)
    else:
        stable = False
    
    # 右侧信号强度分级
    if breakout and expanding and volume and macd_confirm and stable:
        return 'STRONG_RIGHT', '重仓追击（底仓+预备第2批）'
    elif breakout and volume and stable:
        return 'WEAK_RIGHT', '轻仓试探（仅浮动10%）'  # 布林带没开口，可能末端
    else:
        return 'NO_SIGNAL', '等待'
```

### 小周期右侧卖点的标准（移动止盈）

```python
def small_cycle_exit(daily_df, entry_price, current_position):
    # 原则：小周期负责"怎么卖"，但不负责"卖不卖"（大周期红灯才强制清仓）
    
    price = daily_df['close'].iloc[-1]
    profit = (price - entry_price) / entry_price
    
    # 场景1：浮盈30%后，小周期出现顶背离（价格新高，MACD不新高）
    if profit > 0.30:
        price_new_high = price == daily_df['high'].rolling(20).max().iloc[-1]
        macd_not_high = daily_df['macd'].iloc[-1] < daily_df['macd'].iloc[-5]
        
        if price_new_high and macd_not_high:
            return 'TRAIL_EXIT', '移动止盈（浮盈30%+顶背离，减半仓）'
    
    # 场景2：跌破小周期支撑（S1下轨），但大周期仍是绿灯
    # 此时不减底仓，只减浮动仓和预备仓
    if price < daily_df['S1_low'] * 0.98:
        return 'FLOAT_EXIT', '小周期破位，浮动仓止损，底仓持有等周线'
    
    return 'HOLD'
```

## 3. 冲突处理：当小周期给买点，但大周期是黄灯/红灯

这是最容易亏钱的时候，必须机械执行：

| 大周期 | 小周期信号 | **处理方式** | 仓位上限 |
|--------|-----------|-------------|---------|
| **绿灯**（月周向上） | 右侧突破 | **正常执行**（可重仓） | 80% |
| **绿灯** | 无信号/回调 | 等待，不操作 | - |
| **黄灯**（一月一周向上） | 右侧突破 | **降级执行** | 20% |
| **黄灯** | 信号失败 | 立即清仓，不补仓 | 0% |
| **红灯**（任一向下） | 右侧突破（最危险） | **禁止执行**（哪怕突破再漂亮） | 0% |
| **红灯** | S1支撑（抄底诱惑） | **禁止执行** | 0% |

**黄灯特殊规则**：
- 只能做**浮动仓**（10%），不能做底仓或预备仓
- 必须设置** tight 止损**（-5%硬砍，不等待）
- 盈利目标降低（到R1就走，不赌突破）

**红灯特殊规则**：
- **绝对禁止**任何形式的多单
- 如果持有底仓，**禁止**用小周期的"S1支撑"信号补仓（此时S1支撑无效，可能连续破位）
- 只能用小周期的**卖点信号**减仓（右侧卖点变成清仓信号）

- **B点**：波段高点（放量滞涨/长上影）
- **C点**：回调低点（缩量止跌，不破A点）
- **功能**：不决定买卖点，决定**仓位大小**
  - C点与S1重合（共振）→ 重仓（底仓30%+预备第一批8%）
  - C点未确认（B-C途中）→ 轻仓（仅浮动5%日内T）
  - B点附近（M头风险）→ 禁止买入

### 过滤器：布林带（波动率温度计）
- **低位极端收口**（带宽<历史10%）+ S1触及 → 高胜率买点（仓位×1.5）
- **上涨途中收口**（带宽收窄+价格在上轨）→ 防御信号（减持浮动仓，防回调）
- **开口向上**（突破时）→ 确认趋势加速（预备仓第二批17%可入）

```python
    def _synthesize(self, price, zones, structure, bb, df_h4):
        """
        信号合成逻辑（你的核心交易规则量化）
        """
        # 场景A：触及S1支撑带
        if zones['S1_low'] <= price <= zones['S1_high']:
            
            # 检查三点结构：C点是否与S1重合（共振）
            c_near_s1 = abs(structure.get('C', 0) - price) / price < 0.03
            c_confirmed = structure.get('C_confirmed', False)
            
            # 检查布林带：低位收口（高胜率）
            low_squeeze = bb['bandwidth'] < bb['hist_10pct'] and bb['trend'] == 'FLAT'
            
            if c_near_s1 and c_confirmed and low_squeeze:
                return {
                    'signal': 'STRONG_BUY',
                    'confidence': 0.9,
                    'position_type': 'FULL_BUILD',  # 底仓30%+预备8%
                    'reason': 'S1+C点+布林低位收口共振',
                    'zones': zones
                }
            elif c_confirmed:
                return {
                    'signal': 'MODERATE_BUY',
                    'confidence': 0.7,
                    'position_type': 'FLOAT_ONLY',  # 仅浮动10%
                    'reason': 'S1+C点，但布林未收口'
                }
            else:
                return {
                    'signal': 'SCALP',
                    'confidence': 0.5,
                    'position_type': 'DAY_TRADE',  # 日内T
                    'reason': 'S1触及但C点未确认，下跌中继风险'
                }

    def _bollinger_status(self, df):
        """
        布林带状态机
        """
        current = df.iloc[-1]
        prev = df.iloc[-5]
        
        bandwidth = (current['BB_upper'] - current['BB_lower']) / current['BB_middle']
        bandwidth_hist = df['bandwidth'].quantile(0.10)

            else:
                return {'status': 'SQUEEZE_LOWER', 'trend': 'FLAT'}
        
        # 开口判断
        if current['BB_upper'] > prev['BB_upper'] * 1.02:
            return {'status': 'EXPANDING', 'trend': 'UP'}
            
        return {'status': 'NORMAL', 'bandwidth': bandwidth}
```

```python
        geo_score, geo_status = self.geo.calculate_risk_score(
            current_event, user_input_dict
        )
        geo_limit = 1.0
        if self.geo.market_confirmation(data['market']):
            geo_limit, msg = self.geo.get_position_limit(geo_score, True, 1.0)
            if geo_limit < 0.30:
                self.execution.emergency_reduce(geo_limit)  # 强制降仓
        
        # 3. 多周期趋势检查
        monthly_trend = self.tf_filter.get_trend(data['monthly'], 'monthly')
        weekly_trend = self.tf_filter.get_trend(data['weekly'], 'weekly')
        can_trade, tf_limit = self.tf_filter.traffic_light(monthly_trend, weekly_trend)
        
        if not can_trade:
            return {"status": "BLOCKED", "reason": "大周期红灯，禁止做多"}
        
        # 4. 信号生成（只在绿灯或黄灯时运行）
        signal = self.signal_engine.generate_signal(data['daily'], data['h4'])
        
        # 5. 仓位计算
        orders = self.position_engine.calculate_position(
            signal, geo_limit, tf_limit
        )
        
        # 6. 止损检查（独立运行，优先于新信号）
        stop_orders = self.position_engine.check_stop_loss(data['daily']['close'])
        trail_orders = self.position_engine.trailing_stop_base(data['daily']['close'])
        
        # 7. 执行
        all_orders = stop_orders + trail_orders + orders
        for order in all_orders:
            self.execution.submit(order)
            
        return {
            "geo_limit": geo_limit,
            "tf_limit": tf_limit,
            "signal": signal,
            "orders": all_orders
        }
```

```python
        return sorted(zones)  # 自动识别S1, S2, R1, R2
```

### 3. 三点结构识别（替代你的"波段判断"）

**技术方案**：**ZigZag算法 + 形态匹配**

```python
class AI_Swing_Detector:
    def detect_abc_structure(self, df, deviation=0.05):
        """
        自动识别A-B-C三点结构
        """
        from zigzag import peak_valley_pivots
        
        # ZigZag识别显著高低点（过滤噪音）
        pivots = peak_valley_pivots(df['close'], df['high'], df['low'], deviation)
        
        structures = []
        for i in range(len(pivots)-2):
            A = pivots[i]      # 起点
            B = pivots[i+1]    # 高点  
            C = pivots[i+2]    # 回调低点
            
            # 验证结构质量
            ab_return = (B['price'] - A['price']) / A['price']
            bc_retracement = (B['price'] - C['price']) / (B['price'] - A['price'])
            
            if ab_return > 0.20 and 0.382 <= bc_retracement <= 0.618:
                # 验证C点止跌（AI识别或规则）
                c_candle = df.iloc[C['index']]
                if self._is_bounce_candle(c_candle):
                    structures.append({
                        'A': A, 'B': B, 'C': C,
                        'quality_score': self._calculate_quality(A, B, C, df),
                        'target_D': B['price'] + (B['price'] - A['price'])  # 等距投影
                    })
        
        return structures
    
    def _is_bounce_candle(self, candle):
        # AI或规则判断C点是否有效止跌
        lower_shadow = min(candle['open'], candle['close']) - candle['low']
        body = abs(candle['close'] - candle['open'])
        return lower_shadow > body * 2 and candle['volume'] < candle['vol_ma5'] * 0.8
```

```python
        AI或人工输入三个维度
        """
        if user_input:  # 人工模式
            amplification = user_input['amplification']  # 0-10
            impact_scope = user_input['impact_scope']      # 0-10  
            uniqueness = user_input['uniqueness']        # 0-10
            regime_type = user_input['regime_type']        # 'theocratic' adds 10
        else:  # AI自动模式
            amplification = self.ai_analyze_amplification(gp_data)
            impact_scope = self.ai_analyze_cross_market_impact(gp_data)
            uniqueness = self.ai_analyze_historical_uniqueness(gp_data)
            regime_type = self.ai_classify_regime(gp_data)
        
        total = amplification + impact_scope + uniqueness
        if regime_type == 'theocratic':
            total += 10  # 神权国家持久战溢价
            
        self.current_risk_score = total
        return total
    
    def market_confirmation(self, macro_data):
        """市场波动确认（一票否决机制）"""
        checks = {
            'vix_spike': macro_data['vix'] > CONFIG['geo_risk_threshold']['vix_level_min'] and
                        macro_data['vix_change'] > CONFIG['geo_risk_threshold']['vix_spike_min'],
            'oil_spike': macro_data['oil_price_change'] > 0.05,
            'gold_flight': macro_data['gold_price_change'] > 0.03,
            'liquidity_crunch': macro_data['liquidity_index'] > 0.05
        }
        return sum(checks.values()) >= 2  # 至少两个市场确认
    
    def get_position_limit(self, geo_score, market_confirmed):
        """计算强制仓位上限"""
        if not market_confirmed and CONFIG['geo_risk_threshold']['market_confirm_required']:
            return 1.0, "等待市场确认"
            
        if geo_score >= 30:
            return 0.10, "生存模式（1成仓）"
        elif geo_score >= 20:
            return 0.20, "防御模式（2成仓）"
        elif geo_score >= 10:
            return 0.50, "观察模式（5成仓）"
        else:
            return 1.00, "正常模式（100%）"
    
    def emergency_reduce(self, current_positions, target_ratio):
        """紧急降仓执行"""
        excess = sum(current_positions.values()) - target_ratio
        if excess > 0:
            # 优先砍掉预备仓和浮动仓，保留底仓
            reduction_plan = []
            if current_positions['reserve'] > 0:
                cut = min(current_positions['reserve'], excess)
                reduction_plan.append({'layer': 'reserve', 'cut': cut})
                excess -= cut
            if excess > 0 and current_positions['float'] > 0:
                cut = min(current_positions['float'], excess)
                reduction_plan.append({'layer': 'float', 'cut': cut})
            return reduction_plan
        return []

# ==================== 战术层：多周期红绿灯 ====================
class TrafficLightFilter:
    """多周期趋势过滤"""
    
    def analyze_monthly(self, df_monthly):
        """月线趋势"""
        recent_3m_low = df_monthly['low'].iloc[-3:].min()
        prev_3m_low = df_monthly['low'].iloc[-6:-3].min()
        price = df_monthly['close'].iloc[-1]
        ma5 = df_monthly['close'].rolling(5).mean().iloc[-1]
        
        if recent_3m_low < prev_3m_low * 0.98:
            return 'DOWN'
        elif price > ma5 * 1.01:
            return 'UP'
        return 'CONSOLIDATION'
    
    def analyze_weekly(self, df_weekly):
        """周线趋势"""
        # 3周不破新低 + MACD金叉
        recent_3w_low = df_weekly['low'].iloc[-3:].min()
        prev_3w_low = df_weekly['low'].iloc[-6:-3].min()
        macd_signal = calculate_macd(df_weekly)
        
        if recent_3w_low < prev_3w_low * 0.98:
            return 'DOWN'
        elif macd_signal == 'GOLDEN_CROSS' and df_weekly['close'].iloc[-1] > df_weekly['MA20'].iloc[-1]:
            return 'UP'
        return 'CONSOLIDATION'
    
    def get_traffic_light(self, monthly_tf, weekly_tf):
        """红绿灯决策"""
        if monthly_tf == 'UP' and weekly_tf == 'UP':
            return 'GREEN', 1.0
        elif monthly_tf == 'UP' and weekly_tf == 'CONSOLIDATION':
            return 'YELLOW', 0.5
        elif monthly_tf == 'DOWN' or weekly_tf == 'DOWN':
            return 'RED', 0.0
        return 'YELLOW', 0.3

# ==================== 战术层：支撑阻力与三点结构 ====================
class TechnicalAnalyzer:
    """技术分析引擎（支撑阻力+三点结构+布林带）"""
    
    def calculate_support_resistance(self, df_daily):
        """自动计算支撑阻力带"""
        # ZigZag识别显著高低点
        pivots = self.zigzag_pivots(df_daily, deviation=0.05)
        
        # 聚类合并（2%容差）
        support_candidates = [p['price'] for p in pivots if p['type'] == 'low']
        resistance_candidates = [p['price'] for p in pivots if p['type'] == 'high']
        
        s1_cluster = self.dbscan_cluster(support_candidates, eps=0.02)
        r1_cluster = self.dbscan_cluster(resistance_candidates, eps=0.02)
        
        return {
            'S1': (min(s1_cluster) * 0.98, max(s1_cluster) * 1.02) if s1_cluster else None,
            'S2': self.find_second_support(df_daily),  # 更强的历史支撑
            'R1': (min(r1_cluster) * 0.98, max(r1_cluster) * 1.02) if r1_cluster else None,
            'R2': self.find_second_resistance(df_daily)
        }
    
    def detect_abc_structure(self, df_daily):
        """识别三点结构A-B-C"""
        swings = self.zigzag_pivots(df_daily, deviation=0.05)
        
        structures = []
        for i in range(len(swings) - 2):
            A = swings[i]
            B = swings[i+1]
            C = swings[i+2]
            
            # 验证AB涨幅>20%
            ab_return = (B['price'] - A['price']) / A['price']
            if ab_return < 0.20:
                continue
                
            # 验证BC回撤38.2%-61.8%
            bc_retracement = (B['price'] - C['price']) / (B['price'] - A['price'])
            if not (0.382 <= bc_retracement <= 0.618):
                continue
                
            # 验证C点止跌（K线形态）
            c_candle = df_daily.iloc[C['index']]
            if self.is_bounce_candle(c_candle):
                structures.append({
                    'A': A['price'], 'B': B['price'], 'C': C['price'],
                    'C_confirmed': True,
                    'quality_score': self.calculate_structure_quality(A, B, C, df_daily)
                })
        
        return structures[-1] if structures else None  # 返回最新的结构
    
    def is_bounce_candle(self, candle):
        """止跌K线判断"""
        lower_shadow = min(candle['open'], candle['close']) - candle['low']
        body = abs(candle['close'] - candle['open'])
        volume_condition = candle['volume'] < candle['volume_ma5'] * 0.8
        
        return lower_shadow > body * 2 and volume_condition
    
    def bollinger_status(self, df_daily):
        """布林带状态"""
        current = df_daily.iloc[-1]
        bb_width = (current['bb_upper'] - current['bb_lower']) / current['bb_middle']
        bb_width_hist = df_daily['bb_width'].quantile(0.10)
        
        # 判断收口/开口
        prev_upper = df_daily.iloc[-5]['bb_upper']
        expanding = current['bb_upper'] > prev_upper * 1.02
        
        status = {
            'bandwidth': bb_width,
            'is_squeeze': bb_width < bb_width_hist,
            'trend': 'UP' if expanding else 'FLAT',
            'position': 'UPPER' if current['close'] > current['bb_middle'] else 'LOWER'
        }
        
        if status['is_squeeze'] and status['position'] == 'LOWER':
            status['signal'] = 'SQUEEZE_LOWER'  # 低位收口，高胜率买点
        elif status['is_squeeze'] and status['position'] == 'UPPER':
            status['signal'] = 'SQUEEZE_UPPER'  # 上涨收口，防御
        elif expanding:
            status['signal'] = 'EXPANDING'
        else:
            status['signal'] = 'NORMAL'
            
        return status

class SignalEngine:
    """整合所有信号生成交易指令"""
    
    def __init__(self):
        self.tech = TechnicalAnalyzer()
        self.risk_config = CONFIG['risk_management']
        
    def generate_signal(self, symbol, data_dict, geo_limit, tf_limit):
        """主信号生成"""
        price = data_dict['daily']['close'].iloc[-1]
        zones = self.tech.calculate_support_resistance(data_dict['daily'])
        structure = self.tech.detect_abc_structure(data_dict['daily'])
        bb = self.tech.bollinger_status(data_dict['daily'])
        
        # 场景分析
        signal = self._analyze_scenario(price, zones, structure, bb, data_dict['h4'])
        
        # 应用风险限制
        signal['max_position'] = min(geo_limit, tf_limit)
        
        return signal
    
    def _analyze_scenario(self, price, zones, structure, bb, df_h4):
        """场景分析"""
        
        # 场景1：触及S1支撑带
        if zones['S1'] and zones['S1'][0] <= price <= zones['S1'][1]:
            
            # 检查三点结构共振
            c_near_s1 = False
            c_confirmed = False
            if structure:
                c_near_s1 = abs(structure['C'] - price) / price < 0.03
                c_confirmed = structure['C_confirmed']
            
            # 检查布林带
            low_squeeze = bb['signal'] == 'SQUEEZE_LOWER'
            
            if c_near_s1 and c_confirmed and low_squeeze:
                return {
                    'action': 'BUY',
                    'strength': 'STRONG',
                    'position_type': 'FULL_BUILD',  # 底仓30% + 预备第一批8%
                    'entry_price': price,
                    'stop_loss': zones['S1'][0] * 0.97,
                    'target': zones['R1'][0] if zones['R1'] else price * 1.08,
                    'reason': 'S1+C点+布林低位收口共振'
                }
            elif c_confirmed:
                return {
                    'action': 'BUY',
                    'strength': 'MODERATE',
                    'position_type': 'FLOAT_ONLY',  # 仅浮动10%
                    'entry_price': price,
                    'stop_loss': zones['S1'][0] * 0.97,
                    'reason': 'S1+C点，布林未收口'
                }
            else:
                return {
                    'action': 'SCALP',
                    'strength': 'WEAK',
                    'position_type': 'DAY_TRADE',  # 日内T
                    'reason': 'S1触及但C点未确认'
                }
        
        # 场景2：上涨收口防御
        elif price > zones['S1'][1] and bb['signal'] == 'SQUEEZE_UPPER':
            return {
                'action': 'REDUCE',
                'position_type': 'DEFENSE_REDUCE',
                'target_ratio': 0.10,  # 减持浮动至10%
                'reason': '上涨高位收口，防回调'
            }
        
        # 场景3：突破R1右侧追
        elif zones['R1'] and price > zones['R1'][1] * 1.01:
            if bb['signal'] == 'EXPANDING':
                return {
                    'action': 'BUY',
                    'strength': 'BREAKOUT',
                    'position_type': 'PYRAMID_2',  # 预备第二批17%
                    'stop_loss': zones['R1'][1],  # 原阻力变支撑
                    'reason': '突破R1+开口，趋势确认'
                }
        
        return {'action': 'HOLD', 'strength': 'NONE'}

class PositionManager:
    """3-2-5仓位与批次管理"""
    
    def __init__(self, total_capital):
        self.capital = total_capital
        self.layers = {
            'base': {'current': 0, 'cost': 0, 'enabled': False},
            'float': {'current': 0, 'cost': 0, 'enabled': False},
            'reserve': {
                'batch1': {'current': 0, 'cost': 0, 'stop': 0, 'enabled': False},
                'batch2': {'current': 0, 'cost': 0, 'stop': 0, 'enabled': False},
                'batch3': {'current': 0, 'cost': 0, 'stop': 0, 'enabled': False},
                'locked': False  # 第一批止损后锁定
            }
        }
        
    def calculate_orders(self, signal, current_price):
        """根据信号计算订单"""
        orders = []
        max_pos = signal.get('max_position', 1.0)
        
        if signal['action'] == 'BUY':
            orders.extend(self._handle_buy(signal, max_pos, current_price))
        elif signal['action'] == 'REDUCE':
            orders.extend(self._handle_reduce(signal))
        elif signal['action'] == 'STOP':
            orders.extend(self._handle_stop(signal))
            
        return orders
    
    def _handle_buy(self, signal, max_pos, price):
        """处理买入信号"""
        orders = []
        
        if signal['position_type'] == 'FULL_BUILD':
            # 底仓30% + 预备第一批8% = 38%
            target_total = 0.38
            if target_total <= max_pos:
                if self.layers['base']['current'] < 0.30:
                    orders.append(self._create_order('base', 0.30 - self.layers['base']['current'], price))
                if not self.layers['reserve']['batch1']['enabled']:
                    orders.append(self._create_order('reserve_1', 0.08, price))
            else:
                # 超过限制，只建底仓
                if self.layers['base']['current'] < min(0.30, max_pos):
                    orders.append(self._create_order('base', min(0.30, max_pos) - self.layers['base']['current'], price))
                    
        elif signal['position_type'] == 'FLOAT_ONLY':
            # 浮动10%
            if self.layers['float']['current'] < 0.10 and 0.10 <= max_pos:
                orders.append(self._create_order('float', 0.10 - self.layers['float']['current'], price))
                
        elif signal['position_type'] == 'PYRAMID_2':
            # 第二批17%（必须在第一批盈利后）
            batch1 = self.layers['reserve']['batch1']
            if batch1['enabled'] and not self.layers['reserve']['locked']:
                # 检查第一批是否盈利>5%
                pnl = (price - batch1['cost']) / batch1['cost']
                if pnl > 0.05:
                    orders.append(self._create_order('reserve_2', 0.17, price))
                    # 更新第二批止损
                    self.layers['reserve']['batch2']['stop'] = batch1['cost']  # 成本价止损
                    
        return orders
    
    def _handle_reduce(self, signal):
        """处理减仓"""
        orders = []
        if signal['position_type'] == 'DEFENSE_REDUCE':
            # 减持浮动仓至10%
            excess = self.layers['float']['current'] - 0.10
            if excess > 0:
                orders.append(self._create_order('sell_float', excess, market=True))
        return orders
    
    def check_stops(self, current_price):
        """检查止损（独立运行）"""
        stop_orders = []
        
        # 检查浮动仓（S1下轨3%）
        if self.layers['float']['current'] > 0:
            s1_stop = self.memory.get('last_s1_low', current_price * 0.97) * 0.97
            if current_price < s1_stop:
                stop_orders.append(self._create_order('stop_float', self.layers['float']['current'], market=True))
        
        # 检查预备仓阶梯止损
        for i, batch_key in enumerate(['batch1', 'batch2', 'batch3'], 1):
            batch = self.layers['reserve'][batch_key]
            if batch['enabled'] and batch['stop'] > 0:
                if current_price < batch['stop']:
                    stop_orders.append(self._create_order(f'stop_reserve_{i}', batch['current'], market=True))
                    if i == 1:
                        # 第一批止损，锁定后续
                        self.layers['reserve']['locked'] = True
                        stop_orders.append({'action': 'LOCK_RESERVE', 'reason': '第一批止损，禁止后续加仓'})
        
        # 检查底仓移动止盈
        if self.layers['base']['current'] > 0:
            base = self.layers['base']
            high_since_entry = self.get_high_since_entry()
            if high_since_entry > base['cost'] * 1.30:  # 浮盈30%
                if current_price < high_since_entry * 0.90:  # 回撤10%
                    reduce_amount = base['current'] * 0.5  # 减半
                    stop_orders.append(self._create_order('trail_stop_base', reduce_amount, market=True))
        
        return stop_orders

        data = self.data_feed.fetch_market_data(symbol)
        macro = self.data_feed.fetch_macro_data()
        
        # 2. 地缘政治层（最高优先级）
        gp_data = self.data_feed.fetch_geopolitical_feed()
        
        # 检查是否人工输入

            geo_score = self.geo_breaker.calculate_risk_score(gp_data, self.human.manual_geo_input)
        else:
            geo_score = self.geo_breaker.calculate_risk_score(gp_data, None)  # AI模式
            
        market_confirmed = self.geo_breaker.market_confirmation(macro)
        geo_limit, geo_status = self.geo_breaker.get_position_limit(geo_score, market_confirmed)
        
        # 如果地缘限制<30%，立即执行紧急降仓（无论其他信号）
        if geo_limit < 0.30:
            current_pos = self.position_mgr.get_total_position()
            if current_pos > geo_limit:
                reduction = self.geo_breaker.emergency_reduce(current_pos, geo_limit)
                self.execution.execute(reduction)
                log_event(f"地缘熔断触发: {geo_status}, 降仓至{geo_limit}")
                return  # 终止本次交易循环
        
        # 3. 多周期趋势层
        monthly_tf = self.tf_filter.analyze_monthly(data['monthly'])
        weekly_tf = self.tf_filter.analyze_weekly(data['weekly'])
        tf_light, tf_limit = self.tf_filter.get_traffic_light(monthly_tf, weekly_tf)
        
        if tf_light == 'RED':
            log_event("大周期红灯，禁止做多")
            return
        
        # 4. 信号生成层（战术执行）
        max_position_allowed = min(geo_limit, tf_limit)
        signal = self.signal_engine.generate_signal(symbol, data, geo_limit, tf_limit)
        
        if signal['action'] == 'HOLD':
            return
            
        # 5. 止损检查（优先于新开仓）
        stop_orders = self.position_mgr.check_stops(data['daily']['close'].iloc[-1])
        if stop_orders:
            confirmed = self.human.confirm_orders(stop_orders, {'reason': '止损触发'})
            self.execution.execute(confirmed)
        
        # 6. 新仓位计算
        orders = self.position_mgr.calculate_orders(signal, data['daily']['close'].iloc[-1])
        
        # 7. 人工确认（或全自动模式）
        final_orders = self.human.confirm_orders(orders, signal)
        
        # 8. 执行
        if final_orders:
            self.execution.execute(final_orders)
            log_trade(final_orders, signal, geo_status, tf_light)
    
    def run(self):
        """主循环"""
        while market_is_open():
            for symbol in self.symbols:
                self.on_bar_update(symbol, get_current_timestamp())
            sleep(60)  # 分钟级检查
```

1. **分层优先级**：地缘熔断 > 周期过滤 > 技术信号 > 仓位管理

4. **人机混合**：AI处理数据和计算，人类保留地缘判断和否决权
5. **事件驱动**：每根K线触发完整决策链，实时响应

## 一、资金曲线管理机制（Equity Curve Trading）

**你缺失的**：当账户出现连续回撤时，系统没有**自动降低风险敞口**的机制。

**补充方案**：
```python
class EquityCurveManager:
    """资金曲线管理器"""
    
    def __init__(self):
        self.high_water_mark = initial_capital
        self.current_drawdown = 0
        self.risk_levels = [
            (0.00, 0.05, 1.00),    # 回撤0-5%，正常仓位100%
            (0.05, 0.10, 0.70),    # 回撤5-10%，降仓至70%
            (0.10, 0.15, 0.50),    # 回撤10-15%，降仓至50%（进入防御）
            (0.15, 0.20, 0.30),    # 回撤15-20%，降仓至30%（生存模式）
            (0.20, 1.00, 0.10)     # 回撤>20%，强制降至10%或暂停交易
        ]
    
    def adjust_risk_based_on_drawdown(self, current_equity):
        """根据回撤动态调整全局风险系数"""
        self.current_drawdown = (self.high_water_mark - current_equity) / self.high_water_mark
        
        for min_dd, max_dd, risk_factor in self.risk_levels:
            if min_dd <= self.current_drawdown < max_dd:
                return risk_factor
        
        # 更新高点
        if current_equity > self.high_water_mark:
            self.high_water_mark = current_equity
            return 1.00  # 创新高，恢复满仓权限
    
    def apply_to_position_limits(self, base_position_limit):
        """应用到所有仓位决策"""
        # 比如地缘限制给50%，但资金曲线限制给70%，最终取35%
        return base_position_limit * self.risk_factor
```

**关键逻辑**：即使你判断地缘风险低（绿灯），但如果账户已回撤15%，系统应自动将你的**50%仓位上限压缩至25%**（50% × 0.5），防止**"双杀"**（市场跌+你重仓）。

## 二、策略失效检测与切换（Regime Change Detection）

**你缺失的**：当市场结构突变（如从趋势市变为震荡市），系统无法识别**"当前策略已失效"**。

**补充方案**：
```python
class StrategyHealthMonitor:
    """策略健康度监控"""
    
    def __init__(self):
        self.recent_trades = []  # 最近20笔交易记录
        self.baseline_winrate = 0.55  # 系统设计的预期胜率
        
    def detect_regime_change(self):
        """
        检测市场制度是否改变（策略是否失效）
        """
        if len(self.recent_trades) < 20:
            return 'INSUFFICIENT_DATA'
        
        # 指标1：近期胜率暴跌
        recent_winrate = sum([t['profit'] > 0 for t in self.recent_trades[-20:]]) / 20
        if recent_winrate < self.baseline_winrate * 0.60:  # 胜率跌破60%（33%以下）
            return 'DEGRADED'  # 策略退化
        
        # 指标2：盈亏比恶化（赚小亏大）
        avg_win = np.mean([t['profit'] for t in self.recent_trades if t['profit'] > 0])
        avg_loss = abs(np.mean([t['profit'] for t in self.recent_trades if t['profit'] < 0]))
        if avg_win / avg_loss < 1.0:  # 盈亏比<1
            return 'DEGRADED'
        
        # 指标3：连续亏损（马尔可夫链检测）
        consecutive_losses = max([len(list(g)) for k, g in groupby(self.recent_trades, lambda x: x['profit'] > 0) if not k])
        if consecutive_losses >= 5:
            return 'CRITICAL'  # 连续5次亏损，策略可能失效
        
        return 'HEALTHY'
    
    def auto_switch_strategy(self, health_status):
        """自动切换策略模式"""
        if health_status == 'CRITICAL':
            return {
                'action': 'HALT_TRADING',  # 暂停交易1周
                'diagnosis': '策略失效，进入观察期',
                'fallback': '只保留底仓，禁止新开仓'
            }
        elif health_status == 'DEGRADED':
            return {
                'action': 'REDUCE_FREQUENCY',  # 降频交易
                'filter_tightening': '提高信号标准（如需要S1+C点+布林收口三者同时满足才交易）',
                'position_cut': 0.50  # 仓位减半
            }
        return {'action': 'NORMAL'}
```

**场景**：如果你的"三点结构+支撑带"系统连续5次失败（市场进入无趋势混沌期），系统应自动**"冬眠"**（降至1成仓或暂停），而不是继续损耗。

## 三、流动性风险熔断（Liquidity Circuit Breaker）

**你缺失的**：比地缘政治更微观的**个股流动性危机**（闪崩、千股跌停时的无法止损）。

**补充方案**：
```python
class LiquidityRiskManager:
    """流动性风险管理"""
    
    def check_liquidity_before_trade(self, symbol, intended_size):
        """
        交易前检查流动性
        """
        # 指标1：买卖盘深度
        bid_ask_spread = (ask_price - bid_price) / mid_price
        if bid_ask_spread > 0.02:  # 价差>2%
            return False, 'ABORT_SPREAD_TOO_WIDE'  # 放弃交易
        
        # 指标2：冲击成本预估
        daily_volume = get_avg_volume(symbol, 20)
        if intended_size > daily_volume * 0.10:  # 单笔>日成交量10%
            return False, 'ABORT_SIZE_TOO_LARGE'  # 拆分或放弃
        
        # 指标3：跌停板风险（A股特色）
        if is_limit_down(symbol):  # 已跌停
            return False, 'ABORT_LIMIT_DOWN'  # 无法买入，也无法卖出（如果持有）
        
        return True, 'LIQUIDITY_OK'
    
    def emergency_liquidation_protocol(self, portfolio):
        """
        极端流动性危机下的紧急出货协议（比地缘熔断更微观）
        """
        # 当检测到：换手率<1% + 连续3日缩量 + 跌停家数>500家
        if market_liquidity_crunch():
            # 不挂跌停板排队（卖不掉），而是：
            # 1. 跌停板挂卖单（随缘）
            # 2. 同时用股指期货对冲（如果有权限）
            # 3. 或买入反向ETF对冲
            return {
                'action': 'HEDGE_OR_WAIT',
```

**关键**：你现在的止损是**价格止损**，但缺少**流动性止损**（当跌停卖不掉时怎么办）。

```python
class CorrelationMonitor:
    """相关性监控"""
    
    def __init__(self):
        self.positions = {}  # 当前持仓
        self.correlation_matrix = None
        
    def check_concentration_risk(self, new_symbol, new_weight):
        """
        检查新增仓位是否导致行业/风格过度集中
        """
        # 计算现有持仓的行业暴露
        current_exposure = {
            'new_energy': 0.40,  # 当前新能源40%

            'cash': 0.30
        }
        
        # 新标的是否同属新能源
        if get_industry(new_symbol) == 'new_energy':

                return {
                    'allowed': False,
                    'reason': '新能源板块过度集中（>50%），违背分散原则',
                    'suggestion': '选择不同行业标的或降低仓位'
                }
        
        # 计算实际价格相关性（滚动20日）
        for held_symbol in self.positions:

                    'risk_adjustment': 0.50  # 仓位减半
                }
        
        return {'allowed': True}
```
