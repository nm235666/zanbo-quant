## 5. 低分页面深评（重点）

### A) `/signals/graph`（默认态）
- 结论：一般。
- 3 个优点：结构分区清楚；筛选区可控；详情区信息完整。
- 3 个问题：默认态图谱张力弱；首屏“关系图”区域存在感不足；用户不易第一眼理解下一步动作。
- 廉价感来源：大面积浅灰空域 + 主视觉缺少强锚点。
- 产品感来源：筛选、统计、详情、跳转链路完整。
- 一轮优先 5 点：默认示例图；中心节点强化；空态引导文案动作化；关系类型按钮层级再拉开；图谱区微对比增强。

### B) `/chatrooms/investment`
- 结论：一般。
- 3 个优点：业务信息完整；摘要/情绪/标的结构化；筛选操作顺手。
- 3 个问题：卡片内信息密度高；纵向仍偏长；同层级文本权重接近。
- 廉价感来源：多个同样式信息块连续堆叠导致“表单墙”。
- 产品感来源：关键指标、标签、分页动作完整。
- 一轮优先 5 点：卡片主次再分层；弱化次要元信息；默认折叠次级说明；分页与总览更近；增加“高优先群聊”首屏区。

### C) `/system/users`
- 结论：一般。
- 3 个优点：功能闭环全；首屏 KPI 清楚；操作按钮语义明确。
- 3 个问题：用户卡片过高；同屏按钮过多；主操作与次操作视觉对比不够。
- 廉价感来源：大量等权按钮并排。
- 产品感来源：权限/会话/审计同页联动。
- 一轮优先 5 点：用户卡片信息折叠；次要操作下沉菜单；分组表头固定；每卡操作区网格压缩；会话与审计区按优先级默认折叠。

## 6. 本轮遗留清单（确实未完成）
1. `system/users` 仍然纵向偏长。
- 原因：为保持写边界 E2E（依赖默认列表可见目标账号）回退到 `20/页` 默认。
- 影响：全页截图长度仍偏大。
- 下一步：改为“20/页默认 + 分组折叠 + 卡片简版/展开版”双态，不影响测试可见性。

2. `chatrooms/investment` 信息密度仍偏高。
- 原因：业务字段较多，当前仅做了分页与子列表控高。
- 影响：审美分在“可用但不高级”区间。
- 下一步：做卡片信息层级重排（主结论、次指标、可展开明细）。

3. 图谱页默认态产品感仍不足。
- 原因：默认数据经常较稀疏。
- 影响：首次进入时视觉冲击弱。
- 下一步：增加默认案例中心 + 空态引导动作。
