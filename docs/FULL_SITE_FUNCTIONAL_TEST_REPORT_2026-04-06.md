- 市场结论页面 ❌

**发现的问题**:

#### 问题1: 市场结论页面数据为空（核心问题）
- **类型**: 业务正确性问题
- **严重级别**: P1
- **位置**: 市场结论页面（`/app/market`）
- **复现步骤**:
  1. 登录 Admin/Pro 账号
  2. 访问市场结论页
  3. 观察各区块内容
- **实际表现**:
  - 今日交易主线: "暂无足够数据生成今日交易主线摘要"
  - 主要风险: "暂无风险数据"
  - 受益行业: "暂无数据"
  - 受压行业: "暂无数据"
  - 候选方向: "暂无候选方向数据"
  - 数据来源: "暂无来源信息"
- **预期表现**: 应显示基于新闻、信号、宏观数据生成的市场结论
- **影响范围**: 核心投研功能，用户无法获取市场洞察和交易建议
- **根因分析**:
  - 后端代码已添加状态协议（`status`, `status_reason`, `missing_inputs`）
  - 但依赖的数据表不存在或为空：
    - `theme_hotspots` - 不存在（用于受益/受压行业）
    - `investment_signal_tracker` - 不存在（用于候选方向）
    - `macro_series` - 不存在（用于宏观数据）
  - `news_daily_summaries` 存在但 `source_filter` 字段为空
- **建议**:
  1. 创建缺失的数据表
  2. 填充测试数据
  3. 或修改前端展示，明确指引用户如何补充数据

#### 问题2: 决策动作到执行链路不完整
- **类型**: 业务正确性问题
- **严重级别**: P2
- **位置**: 决策板页面（`/app/decision`）
- **实际表现**: 
  - 决策动作记录存在
  - 但"暂无裁决记录"、"暂无人工确认记录"提示
  - 决策到执行的自动映射未完成
- **预期表现**: 决策动作应能自动/手动创建执行任务，形成闭环
- **影响范围**: 决策无法转化为实际执行，业务链路断裂
- **根因**: 
  - 决策动作到执行任务的映射规则未完成
  - `decision_action_id -> order_id -> execution_status` 全链未打通
- **建议**: 明确决策动作类型与执行任务的映射规则，完成状态回写
