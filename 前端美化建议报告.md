## 三、美化建议方案

### 3.1 设计系统优化

#### 建议一：建立三层圆角规范

```css
/* 当前：全部 24-30px */
/* 建议：按层级区分 */
--radius-sm: 12px;   /* 按钮、标签 */
--radius-md: 16px;   /* 卡片、输入框 */
--radius-lg: 24px;   /* 页面级容器、模态框 */
```

**效果**：按钮更紧凑，卡片温和，页面容器保持识别度。

#### 建议二：简化阴影体系

```css
/* 当前：多种阴影混用 */
/* 建议：三层阴影 */
--shadow-soft: 0 2px 8px rgba(0,0,0,0.04);   /* 静态卡片 */
--shadow-card: 0 4px 16px rgba(0,0,0,0.06);  /* 可交互卡片 */
--shadow-float: 0 8px 32px rgba(0,0,0,0.12); /* 悬浮元素、下拉 */
```

**效果**：减少视觉噪点，提升层级清晰度。

#### 建议三：优化颜色对比度

| 元素 | 当前 | 建议 |
|------|------|------|
| 侧边栏背景 | 渐变 `#0d1720 → #0c6977` | 纯色 `#0f172a` 或降低渐变强度 |
| 状态标签 | 浅色底 + 彩色字 | 彩色底 + 白字（提高对比） |
| 禁用状态 | 灰色淡化 | 降低透明度 + 删除线 |

### 3.3 交互体验优化

#### 建议一：全局 Toast 通知

**当前问题**：操作反馈依赖按钮状态变化，不够明显

**方案**：
```typescript
// stores/ui.ts
export const useUiStore = defineStore('ui', () => {
  const toasts = ref<Toast[]>([])
  
  function showToast(message: string, type: 'success' | 'error' | 'info' = 'info') {
    toasts.value.push({ id: Date.now(), message, type })
    setTimeout(() => toasts.value.shift(), 3000)
  }
  
  return { toasts, showToast }
})
```

**使用**：
```vue
<!-- 全局 Toast 容器 -->
<ToastContainer :toasts="ui.toasts" />

<!-- 触发 -->
<button @click="async () => {
  await fetchData()
  ui.showToast('数据已刷新', 'success')
}">
  刷新
</button>
```

#### 建议二：骨架屏 Loading

**当前问题**：数据加载时显示简单文本或空白

**方案**：
```vue
<!-- Dashboard 骨架屏 -->
<div v-if="loading" class="animate-pulse space-y-4">
  <div class="h-8 bg-gray-200 rounded w-1/3"></div>
  <div class="grid grid-cols-3 gap-4">
    <div v-for="i in 6" :key="i" class="h-24 bg-gray-100 rounded-xl"></div>
  </div>
</div>
```

#### 建议三：平滑过渡动画

```css
/* 页面切换 */
.page-enter-active,
.page-leave-active {
  transition: opacity 0.2s ease, transform 0.2s ease;
}
.page-enter-from {
  opacity: 0;
  transform: translateY(8px);
}

/* 卡片悬停 */
.card-hover {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.card-hover:hover {
  transform: translateY(-2px);
  box-shadow: var(--shadow-card);
}
```

## 总结

Zanbo Quant 前端已有良好的视觉基础（渐变、圆角、玻璃拟态），但存在**设计元素过度使用**（渐变、阴影、圆角过多）和**信息层级平均**的问题。建议通过以下方式提升：

1. **简化设计系统**：减少渐变/阴影使用，建立层级规范
2. **重构信息架构**：突出研究重点，折叠运维信息
3. **增强交互反馈**：Toast、骨架屏、过渡动画
4. **优化移动端**：表格转卡片，响应式适配

**优先级**：Dashboard 重构 > 侧边栏简化 > 全局 Toast > 移动端表格
