## 二、问题根因分析

### 2.1 根因一：国内新闻流水线缺少评分步骤 【设计缺陷】

#### 国际新闻流水线（完整流程）
```python
# collectors/news/international.py
commands = [
    {"script": "fetch_news_rss.py"},              # 1. 采集
    {"script": "fetch_news_marketscreener.py"},   # 2. 采集补充
    {"script": "llm_score_news.py"},              # 3. ✓ 评分
    {"script": "llm_score_sentiment.py"},         # 4. ✓ 情绪
    {"script": "map_news_items_to_stocks.py"},    # 5. ✓ 股票映射
]
```

#### 国内新闻流水线（流程缺失）
```python
# collectors/news/domestic.py
commands = [
    {"script": "fetch_cn_news_sina_7x24.py"},     # 1. 采集
    # ❌ 缺少评分步骤！
    # ❌ 缺少情绪分析！
    # ❌ 缺少股票映射！
]
```

**问题**：国内新闻只采集不入即评，完全依赖独立的 `news_sentiment_refresh` 任务（每5分钟）补评分，导致延迟累积。

### 2.2 根因二：评分处理能力不足 【能力缺口】

#### 当前处理能力

| 任务 | 调度频率 | 单次处理 | 理论能力 |
|------|----------|----------|----------|
| news_sentiment_refresh | 每5分钟 | 8条 | 96条/小时 |

#### 实际入库压力

| 来源类型 | 24小时入库 | 平均/小时 |
|----------|-----------|----------|
| 国内新闻 (cn_*) | ~2,647条 | ~110条/小时 |
| 国际新闻 (其他) | ~1,634条 | ~68条/小时 |
| **合计** | **~4,281条** | **~178条/小时** |

**缺口**：处理能力 96条/小时 < 入库速度 178条/小时，存在 **82条/小时** 的能力缺口。

### 2.3 根因三：评分任务模型限制

```python
# llm_score_news.py 默认参数
--limit 30          # 单次最多30条
--sleep 0.3         # 每条间隔0.3秒
--retry 2           # 失败重试2次
```

**瓶颈**：
- 单线程串行处理
- LLM调用延迟（平均2-5秒/条）
- 限流保护（每分钟10-20次）

### 2.4 根因四：特定来源评分失败

| 来源 | 24小时评分率 | 可能原因 |
|------|-------------|----------|
| mw_realtimeheadlines | 0.0% | 内容格式问题/触发异常 |
| mw_marketpulse | 0.0% | 内容格式问题/触发异常 |
| cn_eastmoney_fastnews | 37.3% | 入库时间晚于调度窗口 |

## 三、问题影响评估

### 3.1 直接影响

1. **新闻日报质量下降**：未评分新闻无法进入日报总结
2. **投资信号延迟**：新闻→评分→信号链路延迟2-6小时
3. **用户体验受损**：用户看到"未评分"或"处理中"状态

### 3.2 间接影响

1. **情绪指标失真**：情绪统计不包含最新国内新闻
2. **股票映射滞后**：未评分新闻无法映射到个股
3. **信号时效性**：市场热点反应滞后

## 四、修复方案

### 4.1 短期方案（立即执行）

#### 步骤1：为国内新闻流水线增加评分

修改 `collectors/news/domestic.py`：

```python
from __future__ import annotations

from .common import run_python_commands

NEWS_LLM_MODEL = "deepseek-chat"


def run_cn_news_pipeline() -> dict:
    commands = [
        {"script": "fetch_cn_news_sina_7x24.py", "args": ["--limit", "60", "--timeout", "30"]},
        {"script": "llm_score_news.py", "args": ["--model", NEWS_LLM_MODEL, "--limit", "20", "--retry", "0", "--sleep", "0.05"]},
        {"script": "llm_score_sentiment.py", "args": ["--target", "news", "--model", NEWS_LLM_MODEL, "--limit", "20", "--retry", "0", "--sleep", "0.05"]},
    ]
    return run_python_commands(commands)
```

#### 步骤2：增加国内新闻专用评分任务

在 `job_registry.py` 中添加：

```python
JobDefinition(
    job_key="cn_news_score_refresh",
    name="国内新闻评分补做",
    category="news",
    schedule_expr="*/2 * * * *",  # 每2分钟
    description="为国内新闻补评分，缓解积压",
    commands=(
        py_cmd(str(ROOT_DIR / "llm_score_news.py"), "--model", "deepseek-chat", "--limit", "30", "--source", "cn_sina_7x24"),
    ),
),
```

#### 步骤3：清理当前积压

```bash
# 并行清理积压
bash un_news_score_backlog_parallel.sh --mode cn --parallel 4 --limit-per-source 100
```
